## 記事2
- テーマ: Cloudflare Workers + Expo Pushで「サーバー代0円」のプッシュ通知APIを設計からデプロイまで完全自作する
- ジャンル: プログラミング・技術ノウハウ
- ターゲット読者: サーバー監視・CI/CD・IoTの通知を自前で構築したいエンジニア／Firebase Cloud Messagingのセットアップの複雑さに辟易している開発者／Cloudflare Workersに興味があるが実践的な活用事例を探している人
- 想定価格: ¥780
- 記事の構成案（見出しレベル）:
  1. 完成形のデモ — `curl -X POST` 一発でスマホに通知が届く体験（cURL/Python/Node.jsの3言語サンプル）
  2. アーキテクチャ全体図 — クライアントアプリ → Cloudflare Workers → Expo Push Service → APNs/FCM
  3. なぜFirebaseではなくCloudflare Workersなのか — 無料枠10万リクエスト/日・コールドスタートなし・KVの高速性
  4. 🔒 **===ここから有料===**
  5. Step1: Cloudflare Workers + KVでREST APIを構築する
     - wrangler.tomlの設定全文
     - POST /api/register — Expo Push Tokenとデバイス登録
     - POST /api/send — 優先度(low/normal/high/urgent)・カテゴリ・URL対応の通知送信
     - GET /api/status — 使用量管理と登録確認
     - POST /api/premium — RevenueCat課金状態のサーバー同期
  6. Step2: Expo Push Tokenの取得とデバイス登録
     - registerForPushNotifications()の完全実装（権限リクエスト・Android通知チャンネル設定・projectId指定）
     - registerDevice()によるサーバー登録処理
  7. Step3: 通知受信ハンドラの設計パターン
     - フォアグラウンド受信（addNotificationReceivedListener）
     - getLastNotificationResponseAsync()によるタップ検知 — リスナー未登録パターンの利点
     - syncPendingNotifications: 通知センターからの未読取り込みと重複排除ロジック（extractNotification関数の設計）
     - AppStateリスナーによるアプリ復帰時の自動同期
  8. Step4: APIキー認証とSecureStore
     - generateApiKey()の実装とexpo-secure-storeによる安全な保管
     - AsyncStorageとのフォールバック戦略
  9. 本番運用Tips
     - OTA更新（expo-updates）でストア審査なしにバグ修正をデプロイする方法
     - 使用量制限（月10通フリー）のサーバーサイド実装パターン
     - 応用例: GitHub Actions / Pythonスクリプト / Raspberry Piからの連携
  10. まとめ — このアーキテクチャの月額コストは¥0
- ペイウォール位置の提案: 構成案の4番（Step1の直前）。完成デモ・アーキテクチャ図・技術選定の比較までを無料公開し、「自分もこの構成で作りたい」と思わせる。有料部分ではコピペ可能なTypeScript実装コードをステップバイステップで提供する。無料部分で約4,000字、有料部分で約14,000〜18,000字を想定
- 差別化ポイント:
  - 「プッシュ通知API」をCloudflare Workers（サーバー代0円）で構築する日本語実例はほぼ皆無
  - Expo Push Tokenの取得→サーバー登録→通知送信→受信ハンドリングまで一気通貫のコード（pushService.ts全文ベース）
  - getLastNotificationResponseAsync()による「リスナー未登録パターン」というExpo公式でも紹介が少ない実践テクニック（タップリスナーの二重登録問題を根本解決）
  - 通知の優先度4段階（PRIORITY_CONFIG）・カテゴリ分け・URL付き通知という実用的なAPI設計の解説
  - extractNotification + syncPendingNotificationsによる通知センター同期・重複排除の設計思想
  - cURL/Python/Node.jsの3言語サンプルが記事内に内蔵されており、読者がすぐにターミナルから試せる
