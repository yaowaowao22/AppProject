## 2. 発見した問題・機会

### 【問題1】サブスク疲れ — 「月額¥500のアプリが10個で年間¥60,000」問題
- **JTBD**: 「生産性ツールを使いたい状況で、ユーザーは毎月の出費を気にせず安心して使い続けたい」
- **「不」の分類**: 不安（いつの間にか膨れ上がる固定費への不安）
- **市場シグナル**: 2025年以降「買い切りアプリ」検索が増加。App Storeのサブスク解約率は年間60%超

### 【問題2】ツール分散 — 「メモはNotion、タスクはTodoist、習慣はStreaks…」問題
- **JTBD**: 「複数ツールを横断する状況で、ユーザーは一箇所で自分の行動を把握したい」
- **「不」の分類**: 不便（ツール間の切り替えコスト）
- **市場シグナル**: オールインワンアプリへの回帰トレンド。ただし大手は肥大化して重い

### 【問題3】情報の「あとで読む」墓場 — 保存したが二度と見ない問題
- **JTBD**: 「情報を保存した後の状況で、ユーザーは必要な時に必要な情報を思い出したい」
- **「不」の分類**: 不足（保存→活用の橋渡し機能の欠如）
- **市場シグナル**: Pocket/Instapaperの停滞。ReadwiseはサブスクでiOS買い切り競合なし

### 【問題4】学習の「やった感」問題 — 読書/勉強しても身につかない
- **JTBD**: 「学習した後の状況で、ユーザーは知識が定着していることを確認したい」
- **「不」の分類**: 不満（時間を投資しても成果が見えない）
- **市場シグナル**: Anki系アプリはUIが古い。洗練されたiOSネイティブの間隔反復アプリは少ない

### 【問題5】「何に時間を使ったか分からない」問題
- **JTBD**: 「一日の終わりの状況で、ユーザーは自分の時間の使い方を客観視したい」
- **「不」の分類**: 不快（スクリーンタイムは見せるだけで改善を促さない）
- **市場シグナル**: スクリーンタイムのAPI開放（iOS 15+）。意図的時間管理への需要拡大

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

| 項目 | 内容 |
|------|------|
| **アプリ名案** | TimeGrain（タイムグレイン） |
| **一言コンセプト** | 「時間の粒度」を可視化する — 1日を砂時計のように粒で見せるタイムトラッカー |
| **ターゲットユーザー** | フリーランス・リモートワーカー（25〜40歳）。時間単価を意識しているが、既存のタイムトラッカーは「記録が面倒」で続かない人 |
| **主要機能** | 1. **自動カテゴリ推定** — カレンダー・スクリーンタイムAPIから行動を自動分類（手入力ゼロ） / 2. **砂粒ビジュアライゼーション** — 1日=1440粒。仕事・休息・移動・SNSが色分けされた砂時計UIで直感的に把握 / 3. **週次インサイト** — 「先週より深い仕事が2.3時間増えた」等の自然言語フィードバック |
| **買い切りで売れる理由** | 競合（Toggl, RescueTime）は全てサブスク。「記録するだけなのに月額？」という不満が明確。ローカル処理なのでサーバーコスト不要＝買い切りが持続可能 |
| **想定価格** | ¥980 |
| **差別化ポイント** | ①手入力ゼロの自動記録（逆張り：タイムトラッカーなのにタイマーを押さない）②砂粒UIという新しいメタファー（棒グラフ/円グラフの退屈さから脱却）③完全ローカル処理でプライバシー安心 |
| **解決する課題** | 問題5「何に時間を使ったか分からない」 |
| **実現難易度** | ★★★☆☆（スクリーンタイムAPI + カレンダーAPI連携。砂粒アニメーションにSwiftUI/Metal活用） |
| **市場規模感** | TAM: 時間管理アプリ市場 約$1.5B / SAM: iOS買い切り層 約$50M / SOM: 日本語圏初年度 5,000DL × ¥980 = ¥490万 |
| **思考の起点** | 逆張り：「タイムトラッカーは手動記録が前提」→ 逆に「一切記録しない」トラッカー。Thiel的問い「全員がタイマーボタンを押すべきだと思っているが、本当にそうか？」 |

### 【アイデア3: ◆レンズ — 第一原理（First Principles）】

# 📱 DayStack（デイスタック）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

| 項目 | 内容 |
|------|------|
| **アプリ名案** | DayStack（デイスタック） |
| **一言コンセプト** | 「今日やることは3つだけ」— タスク管理の第一原理に立ち返り、毎日3タスクに強制制限するミニマルプランナー |
| **ターゲットユーザー** | タスク管理ツールを何度も乗り換えては挫折する人（25〜45歳）。Todoistのタスクが100件溜まって開くのが怖い人。完璧主義傾向のある知的労働者 |
| **主要機能** | 1. **3タスク制限** — 1日に登録できるタスクは最大3つ。4つ目を追加するには1つ完了or翌日に送る必要がある。「選ぶ」ことを強制する設計 / 2. **スタック履歴** — 完了タスクが日ごとにカードとして積み上がる。過去30日間の「達成の塔」が成長するビジュアルフィードバック / 3. **朝のコミット儀式** — 毎朝の通知で「今日の3つ」を宣言。Apple Watch対応で手首からコミット＆完了タップ |
| **買い切りで売れる理由** | 機能が「少ない」ことが価値。サブスクで機能追加され続けるツールへのアンチテーゼ。一度買えば永遠にシンプルなまま使えるという安心感が購入動機になる |
| **想定価格** | ¥490 |
| **差別化ポイント** | ①「制限」が機能（競合は機能追加で差別化するが、このアプリは機能削減で差別化）②ゲーミフィケーションではなく「達成の可視化」（ストリークではなくスタック＝途切れても罪悪感なし）③Apple Watch連携で最速のタスク完了体験 |
| **解決する課題** | 問題2「ツール分散」の根本原因 — そもそもタスクが多すぎる |
| **実現難易度** | ★★☆☆☆（SwiftUI + Core Data。Apple Watch対応はWatchKit。シンプルな分、UI磨きに時間を使える） |
| **市場規模感** | TAM: タスク管理アプリ市場 $3B / SAM: ミニマリスト志向iOS層 $20M / SOM: 初年度 12,000DL × ¥490 = ¥588万 |
| **思考の起点** | 第一原理：「タスク管理の本質は何か？」→ 「限られた時間で最重要なことを選び、実行すること」→ 「なぜTodoアプリにタスクが100個入るのか？選ぶ機能がないからだ」→ Musk的分解：タスク管理 = 選択 + 実行 + 記録。選択を強制する仕組みが欠落している |

### 【アイデア4: ◆レンズ — Blue Ocean（新市場創造）】

# 📱 MeetingGhost（ミーティングゴースト）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

| 項目 | 内容 |
|------|------|
| **アプリ名案** | MeetingGhost（ミーティングゴースト） |
| **一言コンセプト** | 「会議の"見えないコスト"を可視化する」— カレンダーと連動し、会議時間を金額換算して表示する意思決定支援アプリ |
| **ターゲットユーザー** | マネージャー・チームリーダー（30〜50歳）。会議が多すぎると感じているが、どの会議を減らすべきか判断基準がない人。リモートワーク中心の企業勤務者 |
| **主要機能** | 1. **コストダッシュボード** — Googleカレンダー/Apple Calendarと連携し、参加者数×時間×時給概算で各会議のコストを自動算出。「今週の会議コスト: ¥347,000」と表示 / 2. **ゴーストスコア** — 各会議に「本当に必要だったか？」を会議後にワンタップ5段階評価。週次で「削減推奨会議TOP3」をレコメンド / 3. **時間奪還レポート** — 月次で「会議を減らしたことで生まれた時間」を可視化。削減のモチベーション維持 |
| **買い切りで売れる理由** | 会議コスト可視化は一度導入すれば継続的に価値を発揮。月額課金するほどの機能量ではないが、¥980の価値は明確にある。「このアプリで会議を1つ減らせば元が取れる」というROI訴求が強力 |
| **想定価格** | ¥980 |
| **差別化ポイント** | ①「会議削減」に特化した唯一のiOSアプリ（カレンダーアプリは予定管理、会議アプリは議事録。削減を目的にしたアプリは存在しない）②ERRCグリッド適用：議事録機能を排除、コスト可視化を創造、スケジュール管理を削減、意思決定支援を引き上げ③「減らす」ことにフォーカスしたBlue Ocean |
| **解決する課題** | 問題5「時間の使い方が分からない」の企業版 |
| **実現難易度** | ★★☆☆☆（EventKit API + シンプルな計算ロジック。UIデザインがキモ。サーバー不要） |
| **市場規模感** | TAM: 会議効率化ツール市場 $2B / SAM: iOS個人購入層 $15M / SOM: 初年度 6,000DL × ¥980 = ¥588万 |
| **思考の起点** | Blue Ocean：ERRCグリッドを会議ツール市場に適用。Eliminate=議事録・録音（Otter.aiがやる）、Reduce=スケジュール管理、Raise=意思決定の質、Create=コスト可視化＋削減レコメンド。Netflix的転換：「会議を管理する」→「会議を減らす」 |

### 【アイデア5: ◆レンズ — SCAMPER変形 + 高速プロトタイプ（Builder）】

# 📱 VoiceDraft（ボイスドラフト）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

| 項目 | 内容 |
|------|------|
| **アプリ名案** | VoiceDraft（ボイスドラフト） |
| **一言コンセプト** | 「声で考え、テキストで仕上げる」— 音声メモを構造化されたドラフト文書に自動変換するライティング支援アプリ |
| **ターゲットユーザー** | ライター・ブロガー・noteクリエイター（25〜40歳）。「書くのは苦手だが話すのは得意」な人。移動中・散歩中にアイデアが浮かぶが、帰宅すると忘れる人 |
| **主要機能** | 1. **ストリーム録音→構造化** — Apple Speech Frameworkでリアルタイム文字起こし。話し終わると自動で「見出し・本文・箇条書き」に構造化。「えーと」「あの」等のフィラー自動除去 / 2. **テンプレート変換** — 構造化テキストを「ブログ記事」「メール」「企画書」「ツイートスレッド」等のテンプレートにワンタップ変換。note投稿用Markdown出力対応 / 3. **差分エディタ** — 音声原文と構造化後を左右に並べて表示。「ここは原文のニュアンスを残したい」部分をタップで復元。AIに丸投げしない安心感 |
| **買い切りで売れる理由** | 音声文字起こしはApple純正APIでオンデバイス処理＝サーバーコストゼロ。構造化ロジックもルールベース+軽量MLでローカル完結。ランニングコストがないから買い切りが自然。競合のAI文字起こしアプリ（月額¥1,000〜）に対し一回¥1,500で永久使用可能 |
| **想定価格** | ¥1,500 |
| **差別化ポイント** | ①SCAMPER「C（組み合わせ）」：ボイスメモ × アウトライナー × テンプレートエンジンの融合。既存アプリはどれか1つだけ②「E（排除）」：クラウド依存を排除。全処理オンデバイスでプライバシー保護③差分エディタで「AI任せ」の不安を解消。人間がコントロールを握る設計 |
| **解決する課題** | 問題2「ツール分散」+ 問題4「アウトプットまでの距離が長い」 |
| **実現難易度** | ★★★☆☆（Speech Framework + NaturalLanguage Framework + SwiftUI。テンプレートエンジンは正規表現+ルールベースでMVP可。2ヶ月で実装可能） |
| **市場規模感** | TAM: 音声入力・文字起こし市場 $3B / SAM: iOS個人ライティング層 $40M / SOM: 初年度 7,000DL × ¥1,500 = ¥1,050万 |
| **思考の起点** | SCAMPER：ボイスメモアプリに「C（組み合わせ）」でアウトライナーを結合、「E（排除）」でクラウド依存を除去、「A（適応）」でnote/ブログ執筆ワークフローに適応。Builder視点：Apple Speech Framework + NaturalLanguage Frameworkの2APIで48時間MVPが可能。Levels的「まず動くものを作り、フィードバックで磨く」 |
