## 1. アプリ概要

### コンセプト
**「時間の粒度」を可視化する** — 1日を砂時計のように粒で見せるタイムトラッカー

### ターゲットユーザー
フリーランス・リモートワーカー（25〜40歳）。時間単価を意識しているが、既存のタイムトラッカーは「記録が面倒」で続かない人。

### 想定価格
¥980（買い切り）

### 解決する課題（JTBD 3種分析）

| JTBD種別 | ステートメント |
|----------|---------------|
| **Functional** | 「1日の時間の使い方を、手間をかけずに正確に把握したい」 |
| **Social** | 「"時間を有効に使っている自分"を客観データで確認し、クライアントやチームに説明できるようにしたい」 |
| **Emotional** | 「何に時間を使ったか分からない漠然とした不安を解消し、コントロール感を得たい」 |

### 逆張りの核心
**「タイムトラッカーなのにタイマーを押さない」**

Thiel的問い「全員がタイマーボタンを押すべきだと思っているが、本当にそうか？」→ 逆に「一切記録しない」トラッカー。

## 2. 主要機能一覧

### MVP機能（初回リリース）

| 機能 | 概要 |
|------|------|
| **自動カテゴリ推定** | カレンダーから行動を自動分類（手入力ゼロ）。予定名の正規表現マッチングでカテゴリを推定 |
| **砂粒ビジュアライゼーション** | 1日=1440粒。仕事・休息・移動・SNSをカテゴリ色分けした砂時計UIで直感的に把握 |
| **週次インサイト** | 「先週より深い仕事が2.3時間増えた」等の自然言語フィードバック。週次サマリー通知 |
| **TimeScore** | 1日の時間の使い方を0〜100点で表現。深い仕事の割合・スイッチング回数の少なさ・計画通り度を加重平均 |

### 将来追加機能

| 機能 | 概要 |
|------|------|
| **クライアントレポートCSV/PDF出力** | プロジェクト別時間集計をCSV/PDF化。請求書の根拠資料として使用（Proクライアントパック ¥490） |
| **砂粒テーマパック** | カラーテーマ5種（ダーク砂漠/サンセット/オーシャン等）。UIパーソナライズ（¥250） |
| **ポモドーロ連携** | 砂粒が25分で落ちきるアニメーション。タイマー機能を内蔵 |
| **注意力バジェット** | 能動的注意（コーディング等）vs 受動的注意（SNSスクロール等）の可視化。砂粒の輝度で注意の質を表現 |
| **ブラインドモード** | 週1レポートのみ表示。日中は砂時計アニメーションだけ表示し、数字を非表示にする |

## 3. 技術要件

> **重要制約①**: Expo Go（React Native + TypeScript）での実装を前提とする
> **重要制約②**: 保守費ゼロ（サーバーレス・ローカル完結）を厳守

### Expo Go制約とフォールバック

| 項目 | 内容 |
|------|------|
| **使用不可API** | DeviceActivityFramework（ScreenTime API）— Expo Go環境では使用不可 |
| **代替手段** | `expo-calendar` でカレンダー連携 + 手動補完UI |
| **フォールバック設計** | カレンダーAPI使用不可時は簡易モードに自動切替え。審査リスクを低減 |

### データ・インフラ

| 項目 | 技術選定 |
|------|---------|
| **ローカルデータ保存** | AsyncStorage / expo-sqlite |
| **プライバシー** | 全データがデバイス内に留まる設計。サーバー通信なし |
| **カレンダー連携** | expo-calendar（EventKit相当）でカレンダー予定取得 |
| **砂粒アニメーション** | React Native アニメーションAPI または expo-gl（SpriteKit相当） |

### 実装スコープ（MVP 48時間概算）

| フェーズ | 実装内容 |
|---------|---------|
| Phase 1 | expo-calendarでカレンダー予定取得 + カテゴリ自動分類（正規表現ベース）|
| Phase 2 | 砂粒UIプロトタイプ（Reanimated / expo-gl 活用）|
| Phase 3 | 1日のタイムライン表示 + 色分けロジック |
| Phase 4 | 週次サマリー画面 + ローカル通知（expo-notifications）|
| Phase 5 | Widget対応（expo-widgets）+ TestFlight/EASビルド |

## 4. UIデザイン要件

### 砂粒UIの仕様

| 項目 | 仕様 |
|------|------|
| **基本単位** | 1日=1440粒（1分=1粒） |
| **メタファー** | 砂時計。上から下に粒が落ちる（時間の経過を視覚化） |
| **レイアウト** | グリッド状に粒を敷き詰める。縦方向に時間軸 |
| **色分けルール** | カテゴリ別に色分け（下記カラーパレット参照） |
| **輝度表現** | 注意力バジェット機能使用時：能動的注意=高輝度、受動的注意=低輝度 |
| **TimeScore連動** | 80点以上の日はゴールド砂粒に変化 |

### カラーパレット案

| カテゴリ | カラー | 用途 |
|---------|--------|------|
| 仕事・深い集中 | 深い青（#1A4B8C等） | コーディング・執筆・会議 |
| 休息・睡眠 | 緑（#2E7D4F等） | 休憩・睡眠・食事 |
| 移動 | オレンジ（#E07B30等） | 通勤・移動時間 |
| SNS・娯楽 | 赤（#C0392B等） | SNS・YouTube等 |
| 未分類 | グレー | カテゴリ推定できなかった時間 |

### ウィジェット要件

- 「飾りたくなる」美しさを最優先
- ホーム画面ウィジェットで今日の砂粒UIをリアルタイム表示
- ロック画面ウィジェットでTimeScoreを表示
- ERRCグリッド方針：ビジュアルの美しさを「引き上げ」、ウィジェットを差別化ポイントに

### 画面一覧

| 画面名 | 概要 |
|--------|------|
| **ホーム（Today）** | 今日の砂粒UI + TimeScore。1440粒グリッドで1日を俯瞰 |
| **週次サマリー** | 7日間のカレンダーにTimeScoreを色付きで表示。「火曜は毎週スコアが低い」等のパターン発見 |
| **設定** | カレンダー権限・カテゴリ色・通知時刻の設定（最小限） |
| **カテゴリ管理** | カテゴリ名・色・キーワードの設定。自動分類の精度補正に使用 |

### UX設計方針

- **「ダウンロードして3秒で使える」体験を最優先**
- オンボーディングなし：カレンダーの権限許可だけで即座にデータが表示される
- 初期設定ゼロ（ERRCグリッド：設定項目数を「削減」）
- 詳細レポートの複雑さを排除し、週次サマリー1画面に集約

## 8. 6レンズブラッシュアップ要約

deepファイル（app-ideas-deep-a.md）から抽出した各レンズの強化案エッセンス。

### レンズ1: First Principles（第一原理）— 「注意力の地図」に再定義

- タイムトラッキングの本質は「何に注意を向けたか」の記録
- 各カテゴリに「能動的注意 vs 受動的注意」のラベルを付与（コーディング=能動的、SNSスクロール=受動的）
- 砂粒の「輝度」で注意の質を表現（将来機能）
- **差別化強化**: 競合は「何に何時間」を記録するが、TimeGrainは「どれだけ集中したか」を記録する唯一のアプリになる

### レンズ2: Contrarian（逆張り）— 「記録を見せない」タイムトラッカー

- 記録はするが、デフォルトでは詳細を見せない。週に一度だけ「時間レポート」が届く設計
- 「ブラインドモード」: 日中は砂時計アニメーションだけ表示、数字は非表示
- **差別化強化**: 「見ないタイムトラッカー」という矛盾したコンセプトが口コミの起点になる

### レンズ3: Connector（異分野接続）— 睡眠トラッカーの「スリープスコア」方式を移植

- AutoSleep/Pillowの「スリープスコア」と同様に「TimeScore」を導入（0〜100点）
- 深い仕事の割合・スイッチング回数の少なさ・計画通り度を加重平均
- TimeScore 80点以上の日はゴールド砂粒に変化
- 月間カレンダーにスコアが色付きで表示→パターン発見（「火曜は毎週スコアが低い」等）

### レンズ4: SCAMPER — Adapt（ポモドーロ統合）+ Put to other use（請求書活用）

- ポモドーロ・テクニックの25分タイマーを砂時計UIに適応（砂粒が25分で落ちきるアニメーション）
- フリーランサーのクライアント請求用途に転用: プロジェクト別時間をCSV/PDFエクスポート
- 「可視化ツール」から「稼ぐツール」へ価値を拡張。¥980の投資を即座に回収できるROI訴求

### レンズ5: Blue Ocean（ERRCグリッド）— 「ダウンロードして3秒で使える」体験

- 排除: 手動タイマー操作、プロジェクト設定画面、チーム共有機能
- 削減: 設定項目数（初期設定ゼロを目指す）、詳細レポートの複雑さ
- 引き上げ: ビジュアルの美しさ（Widgetを「飾りたくなる」レベルに）、プライバシー保証
- 創造: 砂粒UIという新カテゴリ、TimeScore、ゼロ設定の即時利用体験

### レンズ6: Builder（高速プロトタイプ）— 初日課金シナリオ

- ProductHunt Japanで「手入力ゼロのタイムトラッカー」として投稿
- 砂粒UIの30秒デモ動画をX/Twitterに投稿
- 「タイマーを押さないタイムトラッカー」というコピーで初日100DL目標
- App Storeスクショ1枚目に「タイマーを押さないタイムトラッカー」を大きく配置
