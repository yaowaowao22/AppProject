## 1. デザイン哲学 — 鍛鉄の間

### 命名の由来

「鍛鉄（たんてつ）」は鉄を打ち鍛える工程。余分な不純物を叩き落とし、本質だけを残す。「間（ま）」は日本語の空間・間合い・沈黙の概念。この二つを組み合わせた「鍛鉄の間」は、**削ぎ落とすことで生まれる力強い静寂**を意味する。

### マニフェスト

鉄を鍛えるとは、余分なものをすべて削ぎ落とす行為である。高温の炉、鎚の一撃、冷却の静寂。そのサイクルの繰り返しの中にこそ、鋼鉄の純粋性が宿る。このアプリもまた同じ原理で設計されなければならない。ユーザーの思考を奪うあらゆる装飾を取り除き、挙動そのものだけを残す。

骨格（KOKKAKU）が支配する。グリッドは鉄のように正確であり、すべての要素はその骨格の上に配置される。数字は大きく、明確に。重量・セット・レップの記録行為は摩擦ゼロで実現されなければならない。

間（MA）が骨格に息を吹き込む。セットとセットの間の沈黙、余白の積極的活用。Andoが壁の隙間に光を引き込むように、このデザインは余白を主役とする。アプリ自体は空洞の器であり、ユーザーの汗と成長で満たされていく。

色彩は削ぎ落とされ、暗い金属光沢と深い暗色が基調を成し、ひとつの灼熱のアクセントが炉から取り出した赤熱した鉄片のように緊張を与える。

## 2. Ma no Kozo — DDP（Design Direction Package）

### Tension Profile

```
── Tension Profile ──────────────────────────────────
Structure [■■■■□□□□□□] Flow           : 4/10
Silence   [■■■□□□□□□□] Expression     : 3/10  ← ミニマム再設計後

Permanence[■■■■□□□□□□] Impermanence   : 4/10
─────────────────────────────────────────────────────
Dominant layers : KOKKAKU (骨格) + MA (間)
Master references: Fukasawa × Rams × Hara × Ando
```

### Tension 設計理由

| 軸 | 値 | 理由 |
|---|---|---|
| Structure/Flow | 4 | データ入力は構造的、ワークアウトの流れは有機的。どちらも殺さない |
| Silence/Expression | 3 | 日本的抑制。装飾は最小限。オレンジは炉の一点の炎 |
| Precision/Imperfection | 3 | 重量・レップは正確に。ただし冷たい数値UIに堕ちない微細な温もり |
| Universal/Personal | 7 | 鍛錬の哲学は文化的・個人的。グローバル標準UIとは一線を画す |
| Permanence/Impermanence | 4 | 記録は永続する。しかしセッションの高揚感は一瞬の炎 |

### Layer Dominance

- **Primary: KOKKAKU（骨格）** — 鍛錬の規律そのものが骨格。グリッドは鉄のように正確
- **Secondary: MA（間）** — セットとセットの間の静寂。その「間」こそが次の力を生む

### Master References

| デザイナー | 参照する哲学的側面 |
|---|---|
| Naoto Fukasawa | "Without Thought" — インターフェースが消え、行為そのものが残る |
| Dieter Rams | "Less, but better" — 存在理由のない要素は一切存在しない |
| Kenya Hara | 空という器 — アプリはユーザーの成長を受け入れる空洞 |

## 3. カラーシステム

### 設計原則

> オレンジは「炉の一点の炎」。画面上でただ一箇所だけ燃えている。

オレンジ（`#FF6200`）の使用は**3点のみ**に制限:
1. ワークアウト開始ボタン（主CTA）
2. アクティブタブのアイコン＋ラベル
3. セット完了ボタン

それ以外の強調・数値・統計には使わない。

### カラートークン

```css
--bg:   #111113;   /* 背景：鍛冶場の闇 */
--s1:   #191919;   /* 第1表面：ほぼ不可視の浮き上がり */
--s2:   #222224;   /* 第2表面 */
--t1:   #F5F5F7;   /* テキスト主：白に近い */
--t2:   rgba(245,245,247,0.45);  /* テキスト副 */
--t3:   rgba(245,245,247,0.22);  /* テキスト三 */
--ac:   #FF6200;   /* アクセント：灼熱オレンジ */
--acd:  rgba(255,98,0,0.12);     /* アクセントdim */
--ok:   #2DB55D;   /* 成功：セット完了グリーン */
--sep:  rgba(255,255,255,0.07);  /* 区切り線 */
```

### 使用禁止事項

- 統計数値へのオレンジ適用
- 複数箇所へのアクセントカラー分散
- カードへの背景色グラデーション
- ゲーミフィケーション的装飾（炎アイコン乱用・バッジ過剰等）

## 6. UX設計原則

### Action-First 設計

> ユーザーがしたいことを最上位に置く

```
ホーム画面の構造:
1. ワークアウト開始（最大タップターゲット・最上位）
2. 今週の統計（小さく・テキストのみ）
3. クイックスタート
4. 本日のメニュー
```

### トレーニングタブのフロー

```
部位選択（胸/背中/脚/肩/腕/腹）
    ↓
種目リスト（タップ）
    ↓
記録UI（重量 × 回数 × セット完了）
    ↓ スクロール
前回との比較チャート
```

### 情報階層

- **記録中**: UIを忘れる。数字だけが存在する
- **記録後**: 達成感が来る。シートが滑らかに開く
- **振り返り**: 履歴・進捗で成長を感じる

## 7. インタラクション設計

### アニメーション

```css
--sp:  0.3s cubic-bezier(0.34, 1.4, 0.64, 1);  /* Spring: 弾力感 */
--ez:  0.18s ease-out;                            /* Ease: 素直な消退 */
```

| インタラクション | タイプ | 時間 |
|---|---|---|
| タブ切り替え | fade + translateY | 0.18s |
| ボタン押下 | scale(0.97) | 0.3s spring |
| 数値更新 | scale bump | 0.26s spring |
| セット完了 | color → green + scale | 0.36s spring |
| ボトムシート | translateY(100%→0) | 0.3s ease |

### prefers-reduced-motion 対応

アニメーションは `@media (prefers-reduced-motion: reduce)` で全て `0.01ms` に短縮。

## 10. 避けるべきパターン

| アンチパターン | 理由 |
|---|---|
| カラフルなグラデーション背景 | MA（間）の静寂を破壊する |
| オレンジの多用（統計・グラフ等） | 「炉の一点の炎」の希少性が失われる |
| ゲーミフィケーション装飾（炎アイコン・大量バッジ） | 鍛錬の哲学と相反する陳腐さ |
| カード背景の多層化（3層以上） | 空間の密度が上がりMAが死ぬ |
| 「前回を繰り返す」的な受動的UI | ユーザーは能動的に鍛える |
| 挨拶文・励ましコピー | 行為の邪魔。UIは黙って道具であれ |
| `<button>` 内の `<button>` | HTML仕様違反・レイアウト崩壊 |
