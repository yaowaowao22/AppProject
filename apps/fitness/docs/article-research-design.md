## 1. 既存アプリへの不満点（記事の導入に使う）

### 一般的な筋トレアプリのUX問題

**操作フローの摩擦**
- 「種目選択 → テキスト入力欄に重量を打つ → 次のセットを追加 → また打つ → 終わったら手動削除」の繰り返し
- 1セット終わるたびにキーボードを開閉する必要がある
- 入力フォームが縦に並んでいて、セットごとに視線が飛び回る

**情報設計の問題**
- 「今何kg×何repをやろうとしているか」が画面上で目立たない
- 前回の記録が埋もれていて確認しにくい
- 入力完了後の達成感がない（ただ数字を打ち込んだだけ）

**ゲーミフィケーション過剰**
- 炎アイコン・大量バッジ・レベルアップ演出が邪魔
- 「3日連続！おめでとう！」的な励ましコピーが集中を切る
- 本来は「鍛える行為そのもの」が報酬であるはずなのに、外発的なポイントで釣ろうとする

**設計思想の欠如**
- 「なんとなくカラフルにした」グラデーション背景
- オレンジ・赤・青が統計・グラフ・ボタン至る所に氾濫
- UIが前面に出すぎて、ユーザーが「トレーニングをしている」のではなく「アプリを操作している」感覚になる

## 2. TANRENが採用したUX改善策

### 2-1. 重量・回数の「大数字」先出し設計

```
従来: [種目名] [重量___] [回数___] [追加] × 繰り返し
TANREN:
  ┌──────────────────────────────┐
  │         80.0  kg             │  ← 58px Hero Number
  │        [−]        [＋]       │
  │                              │
  │          8   reps            │  ← 58px Hero Number
  │        [−]        [＋]       │
  │                              │
  │  ┌────────────────────────┐  │
  │  │   セット完了（オレンジ）│  │
  │  └────────────────────────┘  │
  └──────────────────────────────┘
```

- 重量・レップ数は58〜64pxの「ヒーローナンバー」で常に上部に表示
- −/＋ボタン（44×44px円形、2.5kg刻み）でタップだけで変更
- 完了ボタンを押すと即反映 → オレンジ→グリーンのアニメーションで完了を体感
- キーボード入力不要。記録中は画面から目を離さなくていい

### 2-2. 部位→種目→セット記録の3ステップフロー

```
Step 1: 部位選択
  胸 / 背中 / 脚 / 肩 / 腕 / 腹
  （2カラムグリッド、背景色なし、区切り線のみ）
      ↓ タップ
Step 2: 種目選択
  ベンチプレス（バーベル）
  ダンベルフライ（ダンベル）
  ...
      ↓ タップ
Step 3: 記録UI
  重量Hero × 回数Hero × セット完了ボタン
  セットログ（完了済みはグリーンチェック）
  前回比較スパークチャート
```

- 各ステップは「戻る」ボタンで前に戻れる
- ホーム画面のクイックスタートチップからはStep 3に直接ジャンプ可能

### 2-3. テンプレートによるクイックスタート

- よく使う種目の組み合わせを「テンプレート」として保存
- テンプレート名 + 種目リストを1タップで展開してワークアウト開始
- TemplateManageScreenで作成・編集・削除が可能
- カスタム種目の追加にも対応（部位・器具種別を指定）

### 2-4. スワイプ操作によるセット削除

- `SwipeableRow` コンポーネントを使用
- セットログ行を左スワイプ → 赤い削除ボタンが現れる → タップで削除
- 誤タップ防止のため「スワイプ」というジェスチャーが削除のトリガー

### 2-5. 前回記録との即時比較

- Step 3 の記録UIにスパークチャートを内蔵
- 過去5〜6セッションの最大重量を棒グラフで視覚化
- 「今日は前回より重い」「停滞している」が一目でわかる
- オレンジ不使用（統計グラフへのアクセントカラー禁止ルール）

## 3. デザイン思想「鍛鉄の間」のエッセンス

### 命名の由来

> 「鍛鉄（たんてつ）」は鉄を打ち鍛える工程。余分な不純物を叩き落とし、本質だけを残す。
> 「間（ま）」は日本語の空間・間合い・沈黙の概念。
> この二つを組み合わせた「鍛鉄の間」は、**削ぎ落とすことで生まれる力強い静寂**を意味する。

### マニフェスト（TANREN_design.md より）

> ユーザーの思考を奪うあらゆる装飾を取り除き、挙動そのものだけを残す。
> 骨格（KOKKAKU）が支配する。グリッドは鉄のように正確であり、すべての要素はその骨格の上に配置される。
> 間（MA）が骨格に息を吹き込む。余白を主役とする。アプリ自体は空洞の器であり、ユーザーの汗と成長で満たされていく。

### Tension Profile

```
Structure [■■■■□□□□□□] Flow          : 4/10
Silence   [■■■□□□□□□□] Expression    : 3/10  ← ミニマム再設計後
Precision [■■■□□□□□□□] Imperfection  : 3/10
Universal [□□□□□□■■□□] Personal      : 7/10
Permanence[■■■■□□□□□□] Impermanence  : 4/10

Dominant layers : KOKKAKU (骨格) + MA (間)
Master references: Fukasawa × Rams × Hara × Ando
```

### 4人のマスター参照

| デザイナー | 参照する哲学 | TANRENへの適用 |
|---|---|---|
| Naoto Fukasawa | "Without Thought" — インターフェースが消え、行為が残る | 記録中はUIを忘れる。数字だけが存在する |
| Dieter Rams | "Less, but better" — 存在理由のない要素は一切存在しない | UIは黙って道具であれ。挨拶文・励ましコピー禁止 |
| Kenya Hara | 空という器 — 受け入れる空洞として機能する | アプリはユーザーの成長を受け入れる空洞 |
| Tadao Ando | 光と構造のリズム — 密度と余白の交互出現 | 密な記録UIと広い履歴画面のコントラスト |

### オレンジは「炉の一点の炎」

> 画面上でただ一箇所だけ燃えている。

オレンジ（`#FF6200` / 実装では `#D4610F`）の使用は**ちょうど3点**に制限:

1. ワークアウト開始ボタン（ホーム画面の主CTA）
2. アクティブタブのアイコン＋ラベル（タブバー）
3. セット完了ボタン（Workout Step 3）

**禁止**: 統計数値、グラフ、PR数値、装飾的バッジへの適用。
「炉の一点の炎」の希少性が、そのオレンジに意味と緊張を与える。

### 画面密度のコントラスト設計

| 画面 | 密度 | 意図 |
|---|---|---|
| アクティブワークアウト | 密 | 集中・緊張感。情報密度を上げる |
| ホーム | 標準 | バランス。CTAが前景 |
| 履歴 | 緩 | 余白広め。達成感・振り返りの開放感 |
| 進捗 | 緩 | データを俯瞰する落ち着いた空間 |

### アンチパターン（設計上の禁止事項）

| パターン | 理由 |
|---|---|
| カラフルなグラデーション背景 | MA（間）の静寂を破壊する |
| オレンジの多用（統計・グラフ等） | 「炉の一点の炎」の希少性が失われる |
| ゲーミフィケーション装飾（炎アイコン・大量バッジ） | 鍛錬の哲学と相反する陳腐さ |
| カード背景の多層化（3層以上） | 空間の密度が上がりMAが死ぬ |
| 受動的UI（「前回を繰り返す」ボタン等） | ユーザーは能動的に鍛える |
| 挨拶文・励ましコピー | 行為の邪魔。UIは黙って道具であれ |

### コントラスト調整スライダー（ThemeContext実装）

`ContrastSettings` として2軸のスライダーを実装:

```typescript
interface ContrastSettings {
  baseLightness: number;    // 0-100、50 = 変化なし（-20〜+20%の明度調整）
  accentSaturation: number; // 0-100、50 = 変化なし（-30〜+30%の彩度調整）
}
```

- 背景・サーフェス・タブバーの明度をリアルタイム調整
- アクセントカラーの彩度を調整（鮮やかさのコントロール）
- AsyncStorageで設定を永続保存

### フォントカスタマイズ（3軸）

```typescript
interface FontSettings {
  fontSizeScale: number;         // 0-100、40 = 等倍(1.0x)、100 = 1.3x
  fontWeightLevel: -1 | 0 | 1;  // 細め / 標準 / 太め
  fontFamily: 'system' | 'serif' | 'mono';
}
```

- **サイズスケール**: 0.80x〜1.30xの範囲でheroNumberから captionまで全サイズが連動
- **ウェイトレベル**: 標準(800/700/600/500) → 細め(600/500/400/300) / 太め(900/800/700/600)
- **フォントファミリー**: system（SF Pro）/ serif（Georgia / serif）/ mono（Menlo / monospace）
- 変更は即座に全画面に反映（useMemo によるリアクティブ再計算）

## 5. 技術スタック概要

### コア技術

| 技術 | 用途 |
|---|---|
| Expo SDK | React Native ビルド基盤・EAS Build |
| React Native | UIフレームワーク |
| TypeScript | 全ソースコードに適用 |
| AsyncStorage | テーマ・フォント設定・ワークアウトデータのローカル保存 |
| React Navigation | 画面遷移（Stack + Tab ナビゲーター）|
| react-native-safe-area-context | SafeArea対応 |
| @expo/vector-icons（Ionicons） | アイコン |

### アーキテクチャパターン

- **Context パターン**: `ThemeContext`（テーマ・フォント）、`WorkoutContext`（ワークアウトデータ）
- **useMemo によるスタイル計算**: `makeStyles(colors, typography)` をメモ化してテーマ切り替えをリアクティブに
- **コンポーネント分離**: `ScreenHeader`、`BottomSheet`、`SwipeableRow` を再利用コンポーネントとして切り出し
- **型安全なデータモデル**: Exercise、WorkoutSet、ExerciseSession、WorkoutDay などをTypeScriptインターフェースで定義

### データ保存

- **AsyncStorage** によるオフライン完結（サーバー不要）
- ワークアウト記録、テーマ設定、フォント設定、コントラスト設定をローカル保存
- カスタム種目もローカルに保存

### AI支援開発

- Claude Code（Anthropic）を使用してコーディング・設計書・仕様書を生成
- `TANREN_design.md`（デザイン思想書）、`docs/spec.md`（機能仕様書）はClaude Codeとの対話で作成
- パッチスクリプト（`patch_ios_submission.py` 等）もClaude Codeで自動生成
