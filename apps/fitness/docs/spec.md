## 1. 画面一覧と機能要件

### タブ構成

| タブID | ラベル | アイコン意味 |
|--------|--------|-------------|
| `home` | 今すぐ | 稲妻（即行動） |
| `workout` | トレーニング | ダンベル |
| `history` | 履歴 | カレンダー |
| `progress` | 進捗 | 折れ線グラフ |

タブは同時に1つだけアクティブ。アクティブタブのアイコン・ラベルのみ `--ac`（オレンジ）を使用。

### 1-1. Home 画面（今すぐ）

**概要**: ユーザーがアプリを開いた直後に最初に見る画面。主CTAとして「ワークアウト開始」を最上位に配置。

#### UI構成（上から順）

| 要素 | 内容 |
|------|------|
| 画面タイトル | 「今すぐ」（26px / bold） |
| 日付表示 | 「2026年3月29日 日曜日」（t3 color / caption） |
| **ワークアウト開始ボタン** | 主CTA（btn-p / オレンジ / 60px高） |
| セクションラベル「今週」 | — |
| 今週統計 | トレーニング回数・kgボリューム・日連続（3列） |
| セクションラベル「クイックスタート」 | — |
| クイックスタートチップ | 横スクロール可能なチップ群 |
| セクションラベル「本日のメニュー」 | — |
| 本日のメニューリスト | 今日記録した種目・最大重量・セット数・PRバッジ |

#### 機能要件

- **ワークアウト開始ボタン**: タップで Workout タブに切り替え（`switchTab('workout')`）
- **今週統計**: 今週のトレーニング回数・総ボリューム（kg）・連続記録日数を表示。数値のみ、オレンジ不使用
- **クイックスタートチップ**: 種目名のショートカット。タップで直接 Workout Step3（記録UI）を開く（`startEx(name, part)`）。デフォルト5件: ベンチプレス/スクワット/デッドリフト/ショルダープレス/懸垂
- **本日のメニュー**: 当日完了した種目の一覧。種目名・最大重量・セット数を表示。PR達成時は「PR」バッジを付与

#### Step 3: 記録UI（ws-active）

| 要素 | 詳細 |
|------|------|
| 戻るボタン | ← Step 2 へ戻る |
| タイマー | セッション開始からの経過時間（MM:SS） |
| 種目名 | 20px/bold |
| 筋肉サブテキスト | 例: 「胸・肩・上腕三頭筋」 |
| セットバッジ | 「セット X/5」（現在セット番号） |
| 重量コントロール | hero number（58px）＋ −/＋ ステップボタン（−2.5/+2.5 kg） |
| 回数コントロール | hero number（58px）＋ −/＋ ステップボタン（−1/+1 rep） |
| **セット完了ボタン** | オレンジ→グリーン遷移 |
| セットログ | セット1〜5の記録行（完了済みはグリーンチェック） |
| 前回比較チャート | 過去5〜6セッションの最大重量スパークチャート |

**重量変化**: 0.0kg以上（0.5刻みではなく2.5刻み）、小数第1位まで表示（例: 80 / 82.5）
**回数変化**: 最小1rep
**セット完了動作**: 完了ボタンが緑（`--ok`）に変化・アニメーション → 1秒後にリセット。5セット完了後は「ワークアウト完了！」表示

### 1-3. History 画面（履歴）

**概要**: 過去のワークアウト一覧をリスト表示。カードタップで日別詳細、種目タグタップで種目履歴を確認できる。

#### UI構成

| 要素 | 詳細 |
|------|------|
| 画面タイトル | 「履歴」 |
| 履歴カードリスト | 日付降順 |

#### 履歴カード（hcard）構造

```
[日付（15px/bold）]                [所要時間（12px/t3）]
[種目タグ 種目タグ 種目タグ]        ← タップで種目詳細シート
[3種目]  [14セット]  [5,820 kg]
```

- カード全体タップ: 日別詳細ボトムシートを開く（`openDaySheet`）
- 種目タグタップ: 種目別履歴ボトムシートを開く（`openExSheet`）、カードタップとは独立（`event.stopPropagation()`）
- 背景なし、区切り線のみ（hcard スタイル）

### 1-4. Progress 画面（進捗）

**概要**: 種目PR・部位別ベスト・週間ボリューム・連続記録・カレンダーを俯瞰する振り返り画面。

#### UI構成

| セクション | 内容 |
|-----------|------|
| 種目PR | 横スクロールPRカード群（オレンジ上線） |
| 部位別1日ベスト | 横スクロール部位カード群（sep上線） |
| 週間ボリューム | 棒グラフ（過去7日間 kg） |
| 連続記録 | 数値大表示（48px） + 過去最高 / 今月目標テキスト |
| 月間カレンダー | 月次カレンダー、ワークアウト実施日にドット、今日を白塗り |

#### 種目PRカード

- 横スクロール
- 種目名（10px/t3/uppercase）・最大重量（30px/bold）・達成日（10px/t3）
- カード上線: `--ac`（オレンジ）1.5px ← Progress画面でのオレンジ使用箇所

#### 部位別1日ベストカード

- 横スクロール
- 部位名（18px/bold）・最大1日ボリューム（22px/bold）・達成日
- カード上線: `--sep`（白7%）
- タップ: 部位概要ボトムシートを開く（`openPartSheet`）

#### 週間ボリュームチャート

- 7日分の棒グラフ（日〜今日）
- 今日の棒: `--t2`（明るいグレー）、過去: `--s2`（暗いグレー）
- **オレンジ不使用**（統計グラフへのオレンジ適用禁止ルール）

#### 月間カレンダー

- トレーニング実施日: ドット表示（`dot` class）、タップで日別詳細シート
- 今日: 白背景+黒文字（`td` class）
- 未来日: 無スタイル

## 2. データモデル定義

### Exercise（種目マスタ）

```typescript
interface Exercise {
  name: string;           // 種目名（例: "ベンチプレス"）
  equipment: EquipmentType; // 器具種別
  part: BodyPart;         // 部位
  muscleDetail?: string;  // 補助筋群テキスト（例: "胸・肩・上腕三頭筋"）
}

type EquipmentType = "バーベル" | "ダンベル" | "マシン" | "ケーブル" | "自重" | "ローラー" | "ツール";
type BodyPart = "胸" | "背中" | "脚" | "肩" | "腕" | "腹";
```

### WorkoutSet（セット記録）

```typescript
interface WorkoutSet {
  setNumber: number;     // 1〜N
  weight: number | null; // kg（自重種目はnull）
  reps: number | null;   // 回数（プランク等はnull）
  isDone: boolean;
}
```

### ExerciseSession（種目別セッション記録）

```typescript
interface ExerciseSession {
  date: string;          // 日付（例: "3/27"）
  exerciseName: string;
  sets: WorkoutSet[];
  maxWeight: number | null; // そのセッションの最大重量
  isPR: boolean;            // このセッションでPR達成
}
```

### WorkoutDay（1日のワークアウト）

```typescript
interface WorkoutDay {
  label: string;         // 表示用日付（例: "3月28日（土）"）
  date: string;          // ISO date（実装時）
  duration: string;      // 所要時間（例: "52分"）
  mainPart: BodyPart;    // メイン部位
  exercises: {
    name: string;
    maxWeight: number | null;
    sets: number;
    isPR: boolean;
  }[];
  totalSets: number;
  totalVolume: number;   // kg（重量 × 回数 の合計）
}
```

### PersonalRecord（種目別PR）

```typescript
interface PersonalRecord {
  exerciseName: string;
  maxWeight: number | null; // null = 自重種目
  achievedDate: string;     // 例: "3/15"
}
```

### PartVolumePR（部位別最大1日ボリューム）

```typescript
interface PartVolumePR {
  part: BodyPart;
  volume: number;  // kg
  date: string;    // 例: "3/27"
}
```

### WeeklyStats（今週統計）

```typescript
interface WeeklyStats {
  trainings: number;  // 今週のトレーニング回数
  volume: number;     // 今週の総ボリューム kg
  streak: number;     // 現在の連続記録日数
}
```

### ExerciseHistoryEntry（スパークチャート用）

```typescript
interface ExerciseHistoryEntry {
  date: string;        // 例: "3/8"（"今日"も許容）
  weight: number | null; // null = 自重
}
```

## 3. 状態遷移

### 3-1. タブナビゲーション

```
         ┌──────────────────────────────────┐
         │           4タブ共通               │
         │  home ←→ workout ←→ history      │
         │           ←→ progress            │
         └──────────────────────────────────┘

switchTab(n) で任意のタブへ即時遷移（アニメーション: fade + translateY, 0.18s）
```

**特記**: Home の「ワークアウト開始」ボタン・クイックスタートチップはどちらも `workout` タブへ遷移する。クイックスタートは追加で Step 3 まで進む。

### 3-2. Workout 3ステップ フロー

```
┌─────────────────┐
│  Step 1          │  初期表示（workoutタブ開時）
│  部位選択        │  ws-parts
│  胸/背中/脚/     │
│  肩/腕/腹        │
└────────┬────────┘
         │ selPart(part)
         ▼
┌─────────────────┐
│  Step 2          │
│  種目選択        │  ws-exlist
│  [← 部位名]     │
│  ・種目A         │
│  ・種目B         │
└────────┬────────┘
         │  startEx(name, part)
         │  ← また Home のクイックスタートチップからも直接ここへ
         ▼
┌─────────────────────────────────────┐
│  Step 3: アクティブワークアウト      │  ws-active
│  [← タイマー]                       │
│  種目名 / 筋肉名                     │
│  セット N/5                          │
│                                     │
│  [重量 hero] [回数 hero]             │
│  [−][＋]     [−][＋]               │
│                                     │
│  ┌───────────────────────────────┐  │
│  │  セット完了（--ac オレンジ）  │  │  ← 2番目のオレンジ使用箇所
│  └───────────────────────────────┘  │
│                                     │
│  セット1: —                          │
│  セット2: —                          │
│  ...                                │
│  セット5: —                          │
│                                     │
│  [前回比較スパークチャート]           │
└─────────────────────────────────────┘
```

#### セット記録のサブ状態遷移

```
[idle]
  ↓ セット完了ボタンタップ
[set_recording] curSet = 1
  ↓ doneSet() 実行
  → セットログ行にテキスト書き込み（"80kg × 8reps"）
  → セット行に done クラス付与（グリーンチェック表示）
  → 完了ボタン: --ok（グリーン）に変化 + "完了！" テキスト
  ↓ 1秒後 setTimeout
  → 完了ボタンをリセット
  → curSet++

[if curSet <= 5]
  → 次のセットへ（set_recording ループ）

[if curSet > 5]  ← 5セット完了後
  → 完了ボタン: "ワークアウト完了！" テキスト + --ok 背景（永続）
  → [workout_complete]

[戻るボタン] → Step 2 へ（状態はリセットされない）
```

### 3-3. ボトムシート

```
[閉じた状態]
  ↓ トリガー（下記3種）
[開いた状態]  overlay + sheet に .on クラス付与
  ↓ overlay タップ or Escape キー
[閉じた状態]

トリガー一覧:
1. openDaySheet(label, dur, part, sets, vol)
   - History の hcard タップ
   - Progress カレンダーの実施日タップ
   → 内容: 日付・部位・時間・ボリューム + 種目ごとの最大重量・セット数・PRバッジ

2. openExSheet(exName)
   - History の種目タグタップ
   - Part概要シート内の種目行タップ
   → 内容: スパークチャート（過去5〜6セッション） + セッション履歴（逆順）+ PRバッジ

3. openPartSheet(part)
   - Progress の部位別ベストカードタップ
   → 内容: 部位の全種目一覧 + 各種目の現PR + 種目行タップで openExSheet へ連鎖
```

## 4. デザイントークン一覧

### カラートークン

| トークン | 値 | 用途 |
|---------|----|----|
| `--bg` | `#111113` | 背景（鍛冶場の闇） |
| `--s1` | `#191919` | 第1表面（カード・ボタン背景） |
| `--s2` | `#222224` | 第2表面（ステップボタン・グラフバー） |
| `--t1` | `#F5F5F7` | テキスト主（白に近い） |
| `--t2` | `rgba(245,245,247,0.45)` | テキスト副 |
| `--t3` | `rgba(245,245,247,0.22)` | テキスト三（ラベル・キャプション） |
| `--ac` | `#FF6200` | アクセント（灼熱オレンジ） |
| `--acd` | `rgba(255,98,0,0.12)` | アクセントdim（PRバッジ背景） |
| `--ok` | `#2DB55D` | 成功（セット完了グリーン） |
| `--sep` | `rgba(255,255,255,0.07)` | 区切り線 |

### オレンジ（`--ac`）使用制限

**画面上でちょうど3箇所のみ**に制限する。

| # | 使用箇所 | 要素 |
|---|---------|------|
| 1 | ワークアウト開始ボタン（btn-p） | Home 主CTA |
| 2 | アクティブタブのアイコン＋ラベル | タブバー |
| 3 | セット完了ボタン（done-btn） | Workout Step 3 |

それ以外での `--ac` 使用は**禁止**（統計数値・グラフ・PRバッジ数字等に適用しない）。
※ PRバッジの背景色は `--acd`（rgba）を使用、文字色も `--ac` だが、これはバッジラベル装飾であり上記3箇所の運用と区別する実装判断が必要。

### アニメーショントークン

| トークン | 値 | 用途 |
|---------|----|----|
| `--sp` | `0.3s cubic-bezier(0.34,1.4,0.64,1)` | Spring（弾力感 / ボタン押下・数値更新） |
| `--ez` | `0.18s ease-out` | Ease（素直な消退 / タブ切り替え・hover） |

| インタラクション | タイプ | 時間 |
|----------------|--------|------|
| タブ切り替え | fade + translateY | 0.18s |
| ボタン押下 | scale(0.97) | 0.3s spring |
| 数値更新（bump） | scale(1.1 → 1) | 0.26s spring |
| セット完了 | bg: --ac → --ok + scale | 0.36s spring |
| ボトムシート開閉 | translateY(100% → 0) | 0.3s ease |
| ヒストリーバー描画 | height | 0.45s cubic-bezier |

`prefers-reduced-motion: reduce` 時は全アニメーション `0.01ms` に短縮。

## 6. アクセシビリティ要件

WCAG 2.2 AA 準拠を目標とする。

| 要件 | 実装 |
|------|------|
| コントラスト比 | 通常テキスト 4.5:1 以上 |
| タップターゲット | 最小 44×44px（stepbtn・ti・cal-d等すべて） |
| フォーカス表示 | `:focus-visible` を全インタラクティブ要素に適用（outline: 2–3px solid `--ac` / `--t3`） |
| モーション | `prefers-reduced-motion: reduce` で全アニメーション `0.01ms` |
| クリッカブル div | `role="button" tabindex="0"` 付与（hcard等） |
| ボタンラベル | `aria-label` を全ボタン・カードに付与（英語UIでも日本語ラベル） |
| ネスト禁止 | `<button>` 内に `<button>` を含めない（hcard内の htag は別要素として実装） |
| ダイアログ | ボトムシートに `role="dialog" aria-modal="true"` |
| タブパネル | `role="tablist"` / `role="tab"` / `role="tabpanel"` / `aria-selected` / `aria-controls` |

## 禁止パターン（アンチパターン）

| パターン | 理由 |
|---------|------|
| カラフルなグラデーション背景 | MA（間）の静寂を破壊する |
| オレンジを3箇所以上に使用 | 「炉の一点の炎」の希少性が失われる |
| 統計数値・グラフへのオレンジ適用 | 同上 |
| ゲーミフィケーション装飾（炎アイコン・大量バッジ） | 鍛錬哲学と相反する |
| カード背景の3層以上の多層化 | MAが死ぬ |
| 受動的UI（「前回を繰り返す」ボタン等） | ユーザーは能動的に鍛える |
| 挨拶文・励ましコピー | UIは黙って道具であれ |
| `<button>` 内の `<button>` | HTML仕様違反・レイアウト崩壊 |
