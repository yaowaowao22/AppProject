# 凛雅 (Ringa) — 日本文化×教養×実用 統一デザイン哲学

> **Ma no Kozo（間の構造）メソドロジーに基づく9アプリ統一デザイン言語**
> 作成日: 2026-03-28 | 対象: 買い切りiOSアプリ群

---

## 哲学名: 凛雅 (Ringa) — Poised Refinement

**「凛」** は冬の朝の空気のように張り詰めた清冽さ。
**「雅」** は宮中文化から受け継がれた洗練と品格。

この二つの文字が表すのは、**無駄を削ぎ落とした緊張感の中にこそ宿る上質さ**である。

凛雅は、日本の生活文化に根ざした9つのアプリケーションに共通する設計思想として生まれた。俳句の十七音に宇宙を詰め込む日本語の凝縮力、茶室の四畳半に無限の精神空間を開く間の技法、そして道具を手にした瞬間に「これでいい」と感じさせる用の美——これらの美意識を、デジタルプロダクトの設計言語として再構築する。

買い切りアプリという形態そのものが、この哲学の体現である。広告の喧騒もサブスクリプションの催促もない静けさ。一度手にすれば、ずっとそこにある道具。それは漆器や和包丁と同じ、**所有する喜びを提供する佇まい**でなければならない。Dieter Ramsの「Less, but better（より少なく、しかしより良く）」は、千利休の「わび」と地下水脈でつながっている。余計なものを削ったあとに残る本質こそが、使い手の人生に寄り添い続ける。

凛雅のデザインは、Massimo Vignelliの「デザインは一つである」という信念を日本文化の文脈で実践する。9つのアプリは異なる機能を持ちながら、同じ骨格、同じ呼吸、同じ声のトーンで語りかける。それはJony Iveが0.1mmの面取りに込めた目に見えない配慮と、Kenya Haraが「白」に見出した無限の可能性を、一つの体系として編み上げたものである。

このデザイン哲学のすべての局面において、**最高峰の職人が幾時間もかけて磨き上げた仕事**であることが、画面の隅々から伝わらなければならない。しかしその手間は、使い手には軽やかさとしてしか感じられない。見える精緻さと感じる自然さの共存——それが凛雅の核心である。

## 2. 六つの深度レイヤー — 適用方針

### Layer 1: MA (間) — 虚の器

> 空間は残りものではない。それは第一の設計素材である。

凛雅において、間は**安土桃山の茶室**に倣う。四畳半という極小空間が宇宙を内包できるのは、すべての間が意味を持つからである。

**全アプリ共通方針:**

- **画面占有率**: コンテンツは画面面積の55〜65%に収める。残りの35〜45%は意図的な余白
- **呼吸点**: 画面をスクロールなしで見たとき、最低1箇所の「呼吸点」（何も置かない領域）を確保
- **間のヒエラルキー**: 余白にも階層がある
  - `ma-xs`: 4pt — 要素内の最小間（テキスト行間調整）
  - `ma-sm`: 8pt — 関連要素間
  - `ma-md`: 16pt — グループ間
  - `ma-lg`: 24pt — セクション間
  - `ma-xl`: 32pt — 大区切り
  - `ma-breath`: 48pt以上 — 呼吸点。何も置かない聖域

**アプリ別適用:**

| アプリ | 間の性格 | 具体適用 |
|--------|----------|----------|
| 俳句帳 | 「枯山水の間」— 広大な余白に一句だけが浮かぶ | 一句表示モード: 画面の70%を余白に。句は中央やや上に配置 |
| 証明写真スタジオ | 「額縁の間」— 写真の周囲を均等に取り囲む | プレビュー画面: 写真の周囲に最低24ptの均等余白 |
| ゴミ分別ガイド | 「目録の間」— リスト項目間の呼吸 | 検索結果: 項目間12pt、セクション間24pt。密すぎず疎すぎず |
| 敬語コーチ | 「試験場の間」— 問題と選択肢の間に思考の余地 | クイズ画面: 問題文と選択肢の間に32ptの思考空間 |
| 冠婚葬祭マナー帳 | 「式場の間」— 格式ある静けさ | リファレンス画面: セクション間32pt。情報密度は低く保つ |
| ワリカン名人 | 「伝票の間」— 数字の読みやすさ優先 | 結果画面: 各人の金額行間に16pt。合計額は上下に32ptの隔離 |
| 御朱印デジタル帳 | 「社寺の間」— 境内の開放感 | コレクション画面: カード間16pt、画面上部に48ptの空 |
| 読み方ドリル | 「稽古場の間」— 集中を促す適度な余白 | 問題画面: 漢字表示エリアと選択肢エリアの間に32pt |
| 四字熟語道場 | 「道場の間」— 畳の目のような規則正しさ | カード画面: 四字熟語は大きく、意味との間に24pt |

### Layer 2: KOKKAKU (骨格) — 不可視の秩序

> すべての要素には構造的理由がある。見る者は明晰さを感じ、グリッドを見ない。

**全アプリ共通の骨格システム:**

```
基本モジュール = 画面高さ ÷ φ (黄金比 1.618)
マージン       = 基本モジュール ÷ 8 = 16pt (iPhone 15基準)
カラム         = 4カラム（コンパクト幅） / 6カラム（レギュラー幅）
ガター         = 8pt
行グリッド     = 8pt
```

**骨格の三層構造:**

1. **大骨格（だいこっかく）**: 画面全体の分割比。黄金比（1:1.618）または白銀比（1:√2 = 1:1.414, 日本の伝統比率）を使用
2. **中骨格（ちゅうこっかく）**: セクション内の配列。4カラムグリッドに従う
3. **小骨格（しょうこっかく）**: 個別コンポーネント内のアライメント。8ptグリッド

**白銀比の採用根拠:**

白銀比（1:√2）は日本のA判用紙・畳・法隆寺の金堂に用いられてきた比率である。西洋の黄金比に対し、白銀比は**やや正方形に近い安定感**を持つ。凛雅では画面分割に白銀比を優先的に使用し、西洋デジタルプロダクトとの視覚的差別化を図る。

**アプリ別の骨格特性:**

| アプリ | 主要比率 | レイアウト方針 |
|--------|----------|----------------|
| 俳句帳 | 白銀比 | 縦書き表示は中央寄せ。一覧は縦スクロールリスト |
| 証明写真スタジオ | 黄金比 | カメラプレビューは画面上部2/3。操作パネルは下部1/3 |
| ゴミ分別ガイド | 白銀比 | 検索バー→結果リストの2分割。フィルターはモーダル |
| 敬語コーチ | 白銀比 | 問題文上部40%、選択肢下部60% |
| 冠婚葬祭マナー帳 | 白銀比 | カテゴリグリッド（2×3）→詳細は全幅スクロール |
| ワリカン名人 | 黄金比 | 入力フォーム上部、結果プレビュー下部 |
| 御朱印デジタル帳 | 白銀比 | 2カラムマソンリー。詳細は全幅フルスクリーン |
| 読み方ドリル | 白銀比 | 問題漢字を大きく中央配置。選択肢は下部2×2グリッド |
| 四字熟語道場 | 白銀比 | フラッシュカード中央。進捗バーは上部固定 |

### Layer 3: SOZAI (素材) — 質感と誠実さ

> デジタルにはデジタル固有の美がある。それでないものを模倣しない。

**原則5「素材に嘘をつかない」の厳格な適用:**

凛雅では、スキューモーフィズム（革のテクスチャ、木目調背景など）を**完全に排除**する。和紙や墨といった伝統素材の質感は、物理素材の模倣としてではなく、**デジタルネイティブな表現として再解釈**する。

**許容される素材表現:**

| 表現 | 解釈 | 使用箇所 |
|------|------|----------|
| 微細なノイズテクスチャ | 和紙の繊維を想起させるが、和紙の写真ではない | 俳句帳・御朱印帳の背景（opacity 3〜5%） |
| ソフトなグラデーション | 墨の濃淡を想起させる階調 | セクション区切りの背景遷移 |
| 1px罫線 | 障子の桟を想起させる繊細な区切り | リスト区切り線（opacity 8〜12%） |
| 角丸の統一 | 12pt統一。陶器の縁の丸みを想起 | すべてのカード・ボタン |
| シャドウ | 極薄（y: 2, blur: 8, opacity: 4%）。浮遊ではなく、わずかな奥行き | カード要素 |

**禁止される素材表現:**

- ❌ 和紙テクスチャの写真を背景に敷く
- ❌ 墨だまり・にじみのビットマップ重ね
- ❌ 竹・木材・石の写真素材
- ❌ リアルな筆書きエフェクト（フォント選択で対応）
- ❌ グラデーションの多用（1画面に2箇所以上は過剰）

### Layer 4: KOKYU (呼吸) — リズムと密度の波

> 吸って（密な塊）、吐いて（開けた空間）をリズミカルに波打たせる。

**全アプリ共通の密度リズム:**

画面を縦にスクロールする体験を、建築のプロムナード（Le Corbusier）として設計する。

```
[入口] → 広い間（ma-breath） → [コンテンツ密] → 間（ma-lg） →
[コンテンツ疎] → 間（ma-xl） → [コンテンツ密] → [余韻の間]
```

**呼吸パターンの3類型:**

1. **俳句型（ゆるやかな呼吸）**: 俳句帳・御朱印デジタル帳・冠婚葬祭マナー帳
   - 密度が低く、間が広い。スクロール全体が「吐く」リズム
   - 一つの情報を大きく見せ、次の情報まで十分な間を取る

2. **問答型（規則的な呼吸）**: 敬語コーチ・読み方ドリル・四字熟語道場
   - 問題→思考→回答→解説の4拍子リズム
   - 各フェーズの密度が明確に異なる（問題: 密→思考: 疎→回答: 密→解説: 中）

3. **道具型（効率的な呼吸）**: 証明写真スタジオ・ゴミ分別ガイド・ワリカン名人
   - 入力→処理→結果の3拍子。最短動線で目的達成
   - 結果表示後に十分な余韻の間を設ける

### Layer 5: KOE (声) — 色彩・書体・感情

> 色と文字は声である。凛雅の声は、低く澄んだ声で語る。

（詳細は後述の「カラーパレット方針」「タイポグラフィ方針」セクションを参照）

**凛雅の声のトーン:**
- 静かだが、明瞭
- 温かいが、なれなれしくない
- 格式があるが、堅苦しくない
- 日本的だが、エキゾチックではない

## 3. 重点原則とアプリ別適用

12の原則から、凛雅のデザイン言語において特に重視する7原則を選定し、アプリ別の具体的適用を示す。

### 原則1: 空の器をデザインせよ

> 空間は残りものではない。それは第一の設計素材である。

**9アプリ共通ルール:**
- 新規画面を設計するとき、**まず余白を配置し、その後にコンテンツを入れる**
- 「ここに何を置くか」ではなく「ここに何を置かないか」から始める

| アプリ | 適用例 |
|--------|--------|
| 俳句帳 | 一句表示画面は、句の周囲に画面の70%相当の余白を確保。余白こそが俳句の世界観を広げる |
| 御朱印デジタル帳 | 御朱印写真の表示画面は、写真1枚を画面の60%に。残り40%は記録情報と余白 |
| 冠婚葬祭マナー帳 | 各マナー項目の間に十分な余白。「急いで読む」ものではなく「落ち着いて参照する」もの |

### 原則4: 制約を選べ

> 少ないタイプフェイス、少ない色、少ない形。制約が発明を生む。

**9アプリ共通の制約:**
- フォントファミリー: **最大2種類**（本文用1 + 表示用1）
- カラー: **基調色1 + アクセント色1 + ニュートラル3段階** = 最大5色
- 角丸: **12pt統一**（例外なし）
- アイコンスタイル: **SF Symbols、ライン型、ウェイト統一**
- アニメーション種別: **最大3種類**（フェードイン、スライド、スプリング）

| アプリ | 制約の活かし方 |
|--------|---------------|
| ワリカン名人 | 色はアンバー系アクセント1色のみ。数字の大小と太さだけで階層を表現 |
| ゴミ分別ガイド | 分別カテゴリは4色のアイコンカラーで識別。それ以外の色は使わない |
| 敬語コーチ | 正解=藍、不正解=朱のみ。それ以外のフィードバック色は存在しない |

## 4. カラーパレット方針 — 和の色彩システム

### 設計思想

日本の伝統色は450色以上あるが、凛雅ではその中から**構造的に選定された最小限のパレット**を使用する。色名は和名を採用し、アプリ全体に文化的一貫性を与える。

### 共通パレット（9アプリ共通）

#### ニュートラル（中性色）— 全アプリの基盤

| 色名 | 用途 | HEX (Light) | HEX (Dark) | 備考 |
|------|------|-------------|------------|------|
| **絹白 (きぬじろ)** | 背景・基調 | `#F7F5F0` | — | 純白(#FFF)は使わない。絹の温かみ |
| **墨白 (すみしろ)** | ダークモード基調 | — | `#1A1A1C` | 純黒(#000)は使わない。墨の深み |
| **灰白 (はいじろ)** | カード背景 | `#EFEDE8` | `#2C2C2E` | |
| **銀鼠 (ぎんねず)** | ボーダー・区切り | `#C8C5BD` | `#48484A` | |
| **鈍色 (にびいろ)** | 二次テキスト | `#8A8680` | `#8E8E93` | |
| **墨 (すみ)** | 本文テキスト | `#2D2926` | `#E8E6E1` | |

#### セマンティックカラー（意味色）— 全アプリ共通

| 色名 | 用途 | HEX | 備考 |
|------|------|-----|------|
| **藍 (あい)** | 正解・成功・リンク | `#1B4965` | 藍染めの深い青 |
| **朱 (しゅ)** | エラー・不正解・警告 | `#C73E1D` | 鳥居の朱 |
| **鶯 (うぐいす)** | 完了・達成 | `#6B7B3A` | 鶯の羽色 |

### アプリ別アクセントカラー

各アプリは共通ニュートラルに加え、**固有のアクセントカラーを1色**持つ。このアクセント色がアプリの個性を決定する。

| アプリ | アクセント色名 | HEX | 由来 | 使用箇所 |

| 敬語コーチ | **藤紫 (ふじむらさき)** | `#6B4C7A` | 高貴さ、ビジネスの格式 | クイズ選択肢ハイライト |

### カラー運用ルール

1. **70-25-5の法則**: 背景ニュートラル70% / テキスト・構造色25% / アクセント色5%
2. **アクセント色の使用制限**: 1画面あたりアクセント色の使用箇所は**最大3箇所**
3. **ダークモード**: 色相は維持し、明度と彩度を調整。背景は墨白(`#1A1A1C`)、テキストは反転
4. **色だけに頼らない**: すべての色情報にはアイコンまたはラベルを併記（アクセシビリティ）
5. **グラデーション制限**: 使用する場合は同一色相内の2段階のみ。多色グラデーションは禁止

## 5. タイポグラフィ方針

### 設計思想

日本語タイポグラフィの美しさは**文字そのものの造形**にある。凛雅ではシステムフォントを基盤としつつ、特定の場面で明朝体を用いることで「和」の品格を表現する。

### フォント体系

```
[表示用 / Display]
  フォント: 游明朝体 (YuMincho) / Hiragino Mincho ProN / serif fallback
  用途:    タイトル、一句表示、四字熟語表示、見出しの格式ある場面
  ウェイト: Demibold (600) — 見出し / Regular (400) — 本文引用

[本文用 / Body]
  フォント: Hiragino Sans / SF Pro (system)
  用途:    本文、UI要素、ラベル、ボタン、説明文
  ウェイト: W3 (300) — 本文 / W6 (600) — 強調 / W7 (700) — 見出し

[数字用 / Numeric]
  フォント: SF Pro (system) — Tabular Figures有効
  用途:    ワリカン名人の金額、スコア、日付、統計値
  特性:    等幅数字（Tabular Figures）で桁揃え
```

### タイポグラフィスケール

8ptグリッドに基づく調和的なスケール:

| レベル | サイズ | 行間 | ウェイト | 用途 |
|--------|--------|------|----------|------|
| Display L | 34pt | 42pt (1.24×) | Demibold | 一句表示、四字熟語大表示 |
| Display M | 28pt | 36pt (1.29×) | Demibold | 画面タイトル |
| Display S | 22pt | 28pt (1.27×) | Bold | セクション見出し |
| Title | 20pt | 26pt (1.30×) | Semibold | カードタイトル |
| Body L | 17pt | 26pt (1.53×) | Regular | 本文（リファレンス系） |
| Body M | 15pt | 24pt (1.60×) | Regular | 本文（標準） |
| Body S | 13pt | 20pt (1.54×) | Regular | 補助テキスト |
| Caption | 11pt | 16pt (1.45×) | Regular | キャプション、タイムスタンプ |

### 縦書き / 横書きの使い分け

| モード | 使用場面 | 技術実装 |
|--------|----------|----------|
| **縦書き** | 俳句帳の一句表示、俳句帳の季語表示、御朱印帳の寺社名 | WebView + `writing-mode: vertical-rl` |
| **横書き** | 上記以外のすべて | React Native標準 |

**縦書き運用ルール:**
- 縦書きは**鑑賞・表示の場面のみ**。入力・操作は常に横書き
- 縦書き表示には明朝体を使用
- 縦書きと横書きが同一画面に混在する場合、明確な領域分離を行う（境界線または余白で区切る）
- 数字・英字は縦中横（`text-combine-upright`）で処理

### アプリ別タイポグラフィ特性

| アプリ | 明朝体の使用 | 特記事項 |
|--------|-------------|----------|
| 俳句帳 | ◎ 多用 | 句の表示は明朝体縦書き。一覧のタイトルも明朝体 |
| 証明写真スタジオ | △ 最小限 | アプリタイトルのみ。UI操作は全てゴシック体 |
| ゴミ分別ガイド | △ 最小限 | 「ゴミ分別ガイド」ロゴのみ。情報検索にはゴシック体 |
| 敬語コーチ | ○ 中程度 | 敬語の例文表示に明朝体。クイズUIはゴシック体 |
| 冠婚葬祭マナー帳 | ○ 中程度 | マナーの引用文・作法名に明朝体 |
| ワリカン名人 | × 不使用 | 数字と操作のみ。全てゴシック体 + SF Pro数字 |
| 御朱印デジタル帳 | ◎ 多用 | 寺社名、参拝日の表示に明朝体 |
| 読み方ドリル | ○ 中程度 | 出題漢字は明朝体大表示。選択肢はゴシック体 |
| 四字熟語道場 | ◎ 多用 | 四字熟語はDisplay Lサイズの明朝体 |

### 間の禁則

1. **余白ゼロの隣接禁止**: いかなる要素も、隣接要素との間に最低 `ma-xs`(4pt) の余白を持つ
2. **均等余白の回避**: 上下左右すべて同じ余白だと「箱に詰めた」感覚になる。**上部の余白を下部より広く**取り、視覚重心を上げる
3. **余白の乱用禁止**: ワリカン名人やゴミ分別ガイドのような道具型アプリでは、余白が多すぎると「操作が遠い」と感じる。道具型は `ma-md` を基準に、必要最小限の間で
4. **呼吸点の必須配置**: スクロールなしで表示される範囲に、最低1箇所の `ma-breath`(48pt+) を必ず含める

## 8. 五段階プロセスの適用ガイド

各アプリの画面を設計するとき、以下のプロセスを必ず経る:

### KEZURU (削る) — 40%を削除

設計した画面の要素を書き出し、40%を削る。残った要素が支配層（MA + KOKKAKU）に奉仕しているか検証する。

### KUMU (組む) — 骨格を先に

白銀比で画面を分割 → 間(ma)を先に配置 → 要素を間の隙間に置く。

### KOMERU (込める) — 声を足す

アクセントカラーを最大3箇所に配置。フォントウェイトで階層を表現。アニメーションは3種から選択。

### TEBANASU (手放す) — 手を離す

原則12のチェックリストで検証。使い手がUIの存在を忘れてコンテンツに没入できるか確認。
