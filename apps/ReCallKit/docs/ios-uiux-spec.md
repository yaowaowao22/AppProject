## 0. エグゼクティブサマリー

ReCallKit は「ゼロ手間保存 × SM-2間隔反復 × 買い切り¥1,200」で Readwise 代替を狙う iOS アプリ。設計思想は **MA（間）** を主軸とし、コンテンツファースト・余白重視・最小限UIクロムを徹底する。Ankiの機能性を Readwise の洗練感で包み、「知識が静かに積み上がる」体験を実現する。

### Tension Profile（暗黙）

| 軸 | 値 | 設計への影響 |
|---|:---:|---|
| Structure / Flow | 4 | 標準タブバー + カードベースレイアウト |
| Silence / Expression | 3 | ミニマルUI・余白重視・色数制限 |
| Precision / Imperfection | 4 | 厳密な8ptグリッド・SF Pro標準 |
| Universal / Personal | 7 | 日本語最適化・高情報密度の適切なバランス |
| Permanence / Impermanence | 3 | 安定した配色・控えめなアニメーション（知識は永続的） |

### Layer Dominance

- **Primary: MA（間）** — 余白重視・コンテンツファースト・装飾抑制
- **Secondary: KOKKAKU（骨格）** — 明確なグリッド・構造的ナビゲーション

## 1. デザイン哲学

### コンセプト: 「静かに積み上がる知識」

ReCallKit のUIは **図書館の書架** をメタファーとする。静かで整然とした空間に、知識が一冊ずつ増えていく感覚。派手な演出ではなく、日々の復習が確実に記憶として定着している実感を、控えめだが確かなフィードバックで伝える。

### 3つの設計原則

1. **フリクションゼロ（Zero Friction）**: Share Extension → 即保存、通知タップ → 即復習。保存も復習も1タップ以内で開始できること
2. **静寂の力（Power of Silence）**: 余白がコンテンツを際立たせる。装飾を排し、保存したテキスト・ハイライトそのものが主役
3. **確実な成長実感（Tangible Growth）**: ストリーク・保存件数・復習完了率を控えめに可視化し、「昨日より賢くなった」実感を提供

## 2. カラーシステム

### アクセントカラー: Recall Amber

知識の温かみ・蓄積感を表現する琥珀色。冷たいブルー系（Readwise）との差別化。

```
Recall Amber
├── Light Mode: #C47F17 (primary actions, accent)
├── Dark Mode:  #F5A623 (primary actions, accent)
└── Usage: ストリーク、進捗、アクティブタブ、CTAボタン
```

### セマンティックカラーマッピング

| 用途 | セマンティック指定 | 補足 |
|---|---|---|
| **Primary Accent** | Asset Catalog `recallAmber` (Dynamic) | Light: `#C47F17` / Dark: `#F5A623` |
| **テキスト（主）** | `.label` | — |
| **テキスト（副）** | `.secondaryLabel` | メタデータ・日時 |
| **テキスト（補助）** | `.tertiaryLabel` | プレースホルダー |
| **背景（主）** | `.systemBackground` | — |
| **背景（副）** | `.secondarySystemBackground` | カード背景 |
| **グループ背景** | `.systemGroupedBackground` | 設定画面 |
| **成功（ストリーク）** | `.systemGreen` / `#30D158` | 連続復習・正解 |
| **警告（期限切れ）** | `.systemOrange` / `#FF9F0A` | 復習遅延 |
| **エラー（Again）** | `.systemRed` / `#FF3B30` | 不正解・リセット |
| **情報** | `.systemBlue` / `#0A84FF` | リンク・ヘルプ |
| **区切り線** | `.separator` | — |

### 色数制限

MA（間）主軸のため、**アクセントカラーは1色（Recall Amber）** に限定。状態表現のみsystemGreen / systemRed / systemOrangeを使用。過度なカラーバリエーションを排除し、余白とタイポグラフィで情報階層を構築する。

### ダークモード

| 要素 | Light | Dark |
|---|---|---|
| 背景（主） | `#FFFFFF` (systemBackground) | `#000000` (systemBackground) |
| 背景（副） | `#F2F2F7` (secondarySystemBackground) | `#1C1C1E` (secondarySystemBackground) |
| カード | `#FFFFFF` + shadow | `#1C1C1E` + border(0.5pt, separator) |
| テキスト（主） | `#000000` (label) | `#FFFFFF` (label) |
| テキスト（副） | `#3C3C43` @60% (secondaryLabel) | `#EBEBF5` @60% (secondaryLabel) |
| Accent | `#C47F17` | `#F5A623` |

## 3. タイポグラフィ

### フォントシステム: SF Pro

学習・知識管理アプリとしての信頼性と可読性を重視し、SF Pro（Rounded ではなく標準）を採用。日本語テキストは SF Pro がヒラギノ角ゴシックにフォールバック。

### タイプスケール

| スタイル | サイズ | ウェイト | 用途 |
|---|---|---|---|
| **Large Title** | 34pt | Bold | 画面タイトル（Today, Library） |
| **Title 1** | 28pt | Bold | セクションヘッダー |
| **Title 2** | 22pt | Bold | カードタイトル |
| **Title 3** | 20pt | Semibold | サブセクション |
| **Headline** | 17pt | Semibold | 強調テキスト・ボタンラベル |
| **Body** | 17pt | Regular | 本文・保存テキスト表示 |
| **Callout** | 16pt | Regular | 補足説明 |
| **Subheadline** | 15pt | Regular | メタデータ（日時・タグ） |
| **Footnote** | 13pt | Regular | 注釈・ヘルパーテキスト |
| **Caption 1** | 12pt | Regular | タブバーラベル・バッジ |
| **Caption 2** | 11pt | Regular | 最小テキスト |

### Dynamic Type 対応（必須）

- 全テキストは `UIFont.preferredFont(forTextStyle:)` 相当で指定
- React Native実装: `allowFontScaling={true}` をデフォルト化
- 固定ptサイズ指定は禁止

### 日本語テキスト考慮

- 行間（lineHeight）: Body テキストで 1.6〜1.8 倍（英語 1.4〜1.5 倍より広め）
- 折り返し: 禁則処理対応（句読点が行頭に来ない）
- カード内テキスト: 最大3行表示 + "..." truncation

## 5. ナビゲーションアーキテクチャ

### タブバー構成（4タブ）

```
┌────────────┬────────────┬────────────┬────────────┐
│  📅 Today  │  📚 Library │  🗺 Map    │  📓 Journal │
│  今日      │  ライブラリ │  マップ    │  ジャーナル │
└────────────┴────────────┴────────────┴────────────┘
```

| # | タブ | SF Symbol | ラベル | 役割 |
|---|---|---|---|---|
| 1 | **Today** | `calendar.day.timeline.left` | 今日 | ホーム。今日の復習カード・ストリーク |
| 2 | **Library** | `books.vertical` | ライブラリ | 保存アイテム一覧・検索・フィルター |
| 3 | **Map** | `point.3.connected.trianglepath.dotted` | マップ | ナレッジマップ（関連グラフ） |
| 4 | **Journal** | `book.closed` | ジャーナル | 学びジャーナル・振り返り |

### タブバー仕様

- アクティブ: Recall Amber（ソリッドアイコン）+ ラベル
- 非アクティブ: `.secondaryLabel`（ラインアイコン）+ ラベル
- ラベルテキスト必須（アイコンのみ禁止）
- タップターゲット: 各タブ最小 44×44pt

### 設定へのアクセス

Today画面のナビゲーションバー右端に `gearshape` アイコン配置。設定は独立タブではなくモーダルプッシュ遷移。

### 画面遷移フロー

```
Today
 ├── → Review（フルスクリーンモーダル）
 │     └── 難易度選択 → 次カード → 完了サマリー
 ├── → QuizMode（フルスクリーンモーダル）
 │     └── 穴埋めクイズ → 正解/不正解 → 結果サマリー
 └── → Settings（プッシュ遷移）

Library
 ├── → Item Detail（プッシュ遷移）
 │     └── 編集 / タグ管理 / 削除
 └── → Search Results（インライン）

Map
 └── → Node Detail（ボトムシート .medium）

Journal
 ├── → Journal Entry Detail（プッシュ遷移）
 └── → Calendar ⇄ List 切替（インライン）
```

### ジェスチャー

| ジェスチャー | 動作 | 画面 |
|---|---|---|
| エッジスワイプ（左端→右） | 前画面に戻る | 全Push遷移画面 |
| 下スワイプ | モーダルを閉じる | Review, QuizMode, Settings |
| 左スワイプ（カード上） | カードをフリップ | Review画面 |
| ピンチイン/アウト | ズーム | ナレッジマップ |
| パン | マップ移動 | ナレッジマップ |

## 13. エラーハンドリング

### エラーメッセージ 3要素テンプレート

| エラー場面 | 何が起きたか | なぜ起きたか | どうすればいいか |
|---|---|---|---|
| **保存失敗** | "保存できませんでした" | "ストレージ容量が不足しています" | "不要なデータを削除してから再度お試しください" |
| **OCR失敗** | "テキストを読み取れませんでした" | "画像が不鮮明な可能性があります" | "より鮮明なスクリーンショットで再度お試しください" |
| **IAP失敗** | "購入を完了できませんでした" | "App Storeとの通信に問題がありました" | "しばらくしてからもう一度お試しください" |
| **iCloud同期失敗** | "バックアップできませんでした" | "iCloudの空き容量が不足しています" | "iCloudストレージを確認してください" |
| **通知許可なし** | — | — | 設定画面に "通知がオフです。設定アプリから有効にしてください" バナー表示 |

### エラー表示方式

- **インラインエラー:** フォーム入力のバリデーション → フィールド下部に赤テキスト
- **トーストバナー:** 一時的エラー → 画面上部にスライドイン、3秒後に自動消去
- **アラート:** 破壊的操作の確認 → 標準アラートダイアログ（ボタン2つまで）
- **空状態フォールバック:** データ取得失敗 → 空状態デザインで表示 + リトライボタン
