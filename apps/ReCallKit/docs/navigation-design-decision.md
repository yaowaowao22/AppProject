# ReCallKit — HIG準拠ナビゲーション設計決定書

> 作成日: 2026-03-31
> ステータス: **決定済み**
> 入力: `docs/analysis/ios-hig-navigation-analysis.md`（HIG分析レポート）
> 制約: サイドバー機能は必須（タグ・コレクション・知識カテゴリの構造的ナビゲーション）

---

## 1. 設計課題

ReCallKitはタグ・コレクション・知識カテゴリによる構造的ナビゲーション（サイドバー機能）を必須としている。しかし、HIG分析レポートにより以下の乖離が確認された:

| 乖離 | 重大度 |
|------|:------:|
| iPhoneでの左ドロワーサイドバー | **高** |
| 左端エッジスワイプとiOS「戻る」ジェスチャーの競合 | **高** |
| ハンバーガーメニューの使用 | **中** |
| タブバー + ドロワーの二重ナビゲーション | **中** |

**課題:** サイドバーが提供する機能（知識体系の構造的ナビゲーション）をHIG準拠パターンで実現する方法を決定する。

---

## 2. 設計決定

### 決定方針: プラットフォーム適応型サイドバー

サイドバー機能をデバイスごとに最適な形態で提供する。iPhoneでは画面内フィルタUI、iPadでは常時表示サイドバー。

## 3. iPhone — Library統合フィルタUI（代替案D採用）

### 3.1 概要

HIG分析の代替案D（ハイブリッド方式）を採用する。サイドバーが担っていたフィルタ機能をLibrary画面内に統合し、頻用フィルタは常時表示、高度なフィルタはボトムシートに配置する。

### 3.2 廃止するパターン

| 廃止パターン | 理由 |
|------------|------|
| 左ドロワーサイドバー | HIG非推奨。発見性の著しい低下 |
| ハンバーガーメニューアイコン（≡） | Apple純正に前例なし。片手操作困難 |
| 左端エッジスワイプ（サイドバー開閉） | iOS標準「戻る」ジェスチャーと競合 |

### 3.3 Library画面ワイヤーフレーム

```
┌──────────────────────────────────────┐
│ Nav: "ライブラリ" (Large Title) 🔍 ⫶  │  ← ⫶ = line.3.horizontal.decrease.circle
│──────────────────────────────────────│
│  [すべて | URL | テキスト | スクショ] │  ← 常時表示セグメントチップ
│  [フロントエンド] [DB] [心理学] [+3]  │  ← 常時表示タグチップ（横スクロール）
│──────────────────────────────────────│
│                                      │
│  今日 ──────────────────────────     │
│  ┌────────────────────────────────┐  │
│  │ 🔗 TypeScriptの型ガード完全解説│  │
│  │    zenn.dev · #フロントエンド   │  │
│  │    次回復習: 4月2日             │  │
│  └────────────────────────────────┘  │
│                                      │
│  （アイテム一覧続き）                 │
│                                      │
│ ──────────────────────────────────── │
│  📅 Today  📚 Library  🗺 Map  📓    │
└──────────────────────────────────────┘
```

### 3.4 詳細フィルタボトムシート

ナビゲーションバー右端のフィルタアイコン（`line.3.horizontal.decrease.circle`）タップで表示。

```
┌──────────────────────────────────────┐
│ ─ ─ ─（detent: .medium）─ ─ ─ ─ ─   │
│ ┌──────────────────────────────────┐ │
│ │ フィルタ              [リセット] │ │
│ │                                  │ │
│ │ 種類:                            │ │
│ │ ○ すべて  ○ URL  ○ テキスト      │ │
│ │ ○ スクリーンショット             │ │
│ │                                  │ │
│ │ コレクション:                    │ │
│ │ [技術メモ] [読書メモ] [仕事]     │ │
│ │                                  │ │
│ │ タグ:                            │ │
│ │ [フロントエンド✓] [DB] [心理学]  │ │
│ │ [プログラミング] [デザイン]      │ │
│ │                                  │ │
│ │ 復習状態:                        │ │
│ │ ○ すべて  ○ 復習待ち  ○ 完了     │ │
│ │                                  │ │
│ │ 日付範囲:                        │ │
│ │ [過去7日] [過去30日] [すべて]    │ │
│ │                                  │ │
│ │      [適用する]                  │ │
│ └──────────────────────────────────┘ │
└──────────────────────────────────────┘
```

### 3.5 フィルタUI仕様

| 要素 | 仕様 |
|------|------|
| **種類セグメント** | 横スクロールChip。選択中: Recall Amber背景 + 白テキスト。非選択: `.systemGray5` + `.label`。高さ32pt、角丸16pt |
| **タグチップ** | 横スクロール。選択中: Recall Amber背景。末尾"+N"で折りたたみ。使用頻度順にソート |
| **フィルタアイコン** | SF Symbol: `line.3.horizontal.decrease.circle`。フィルタ適用中は `.fill` バリアント + Recall Amber |
| **ボトムシート** | `.presentationDetents([.medium, .large])`。`.medium`で開始。ドラッグで`.large`へ拡張可能 |
| **適用中表示** | フィルタ適用中はナビゲーションバー下にサマリーチップ表示。タップで個別解除可能 |

### 3.6 ジェスチャー設計（競合解消）

| ジェスチャー | 動作 | 備考 |
|---|---|---|
| 左端エッジスワイプ（左→右） | **前画面に戻る（iOS標準）** | サイドバー開閉には使用しない |
| 下スワイプ | モーダル/シートを閉じる | Review, Quiz, ボトムシート |
| 横スクロール | チップ一覧スクロール | 種類・タグフィルタ |

## 4. iPad — TabView Sidebar統合

### 4.1 概要

iPadOS 18+の`TabView` Sidebar統合を採用する。同一のSwiftUIコードベースで、iPhoneではタブバー、iPadではサイドバーとして自動変換される。

### 4.2 iPad (Regular) レイアウト

```
┌─────────────┬──────────────────────────────────┐
│  Sidebar     │  Main Content                    │
│  (280pt)     │                                  │
│              │                                  │
│  ● Today     │  ┌─────────────────────────────┐ │
│  ● Library   │  │  選択された画面の内容        │ │
│    ├ すべて  │  │                             │ │
│    ├ コレクション│ │                           │ │
│    │ 技術メモ │  │                             │ │
│    │ 読書メモ │  │                             │ │
│    │ 仕事    │  │                             │ │
│    └ タグ    │  │                             │ │
│      #フロントエンド│                          │ │
│      #DB     │  │                             │ │
│      #心理学 │  │                             │ │
│  ● Map       │  │                             │ │
│  ● Journal   │  │                             │ │
│              │  └─────────────────────────────┘ │
│──────────────│                                  │
│  ⚙ Settings  │                                  │
└─────────────┴──────────────────────────────────┘
```

### 4.3 Sidebar適用: DDP "Marginal Silence" 哲学

iPad Sidebarには`ddp-sidebar.md`の「余白の沈黙」哲学をフル適用する:

| DDP原則 | iPad Sidebar適用 |
|---------|-----------------|
| **余白が主役** | 項目間の垂直間隔を十分に確保。8ptグリッドの整数倍でリズムを構築 |
| **骨格の不可視性** | セクション区切りは間隔の変化のみで表現。罫線・背景色分けは不使用 |
| **光のスリット** | アクティブ項目のみRecall Amberでハイライト。他は沈黙 |
| **空の器** | 初期状態は4タブのみの最小構成。ユーザーのタグ・コレクションが育つにつれ充実 |
| **知識の背骨** | タグ階層がそのまま知識マップ。コレクション配列が関心の年輪 |

### 4.4 デバイス対応まとめ

| デバイス | Size Class | ナビゲーション形態 | サイドバー機能 |
|---------|-----------|-----------------|-------------|
| iPhone | Compact | ボトムタブバー（4タブ） | Library画面内フィルタUI |
| iPad (Portrait) | Compact | ボトムタブバー（4タブ） | Library画面内フィルタUI |
| iPad (Landscape) | Regular | TabView Sidebar | 常時表示サイドバー |
| iPad (Split View) | Compact/Regular | 幅に応じて自動切替 | 自動切替 |

## 5. 実装方針

### 5.1 技術スタック対応（Expo + React Native）

| 要素 | 実装手段 |
|------|---------|
| タブバー | `@react-navigation/bottom-tabs` — 現行維持 |
| Library画面内チップ | `ScrollView` horizontal + `Pressable` Chip コンポーネント |
| ボトムシート | `@gorhom/bottom-sheet` — iOS標準のdetent挙動を再現 |
| iPad Sidebar | Phase 2。`react-native-drawer-layout` + Size Classによる条件分岐、またはExpo Router v4のTabView Sidebar対応時に移行 |

### 5.2 Phase対応

| Phase | 対象 |
|-------|------|
| **Phase 1（MVP）** | iPhone 4タブ + Library画面内フィルタUI。ボトムシートは種類・タグフィルタのみ |
| **Phase 1.5** | ボトムシートにコレクション・復習状態・日付範囲フィルタを追加 |
| **Phase 2** | iPad Regular Width でのTabView Sidebar。DDP "Marginal Silence" フル適用 |

## 6. DDP "Marginal Silence" の継承先

`ddp-sidebar.md`のデザイン哲学は以下のように継承される:

| DDP哲学 | iPhone（Library Filter UI） | iPad（Sidebar） |
|---------|---------------------------|----------------|
| 余白の沈黙 | チップ間・セクション間の間隔設計 | サイドバー項目間の垂直リズム |
| 骨格の不可視性 | フィルタUIの8ptグリッド構造 | セクション区切りの間隔変化 |
| 光のスリット | 選択チップのRecall Amberハイライト | アクティブ項目のAmberハイライト |
| 2ウェイトのタイポグラフィ | セクション見出し(Semibold) + チップ(Regular) | 同左 |
| 最小限の色数 | 背景/テキスト/Amber の3層 | 同左 |

---

## 7. 決定根拠まとめ

| 判断 | 根拠 |
|------|------|
| iPhone左ドロワー廃止 | HIG非推奨。Apple純正に前例なし。発見性・ジェスチャー競合の重大問題 |
| 代替案D（ハイブリッド）採用 | 頻用操作は1タップ、高度操作はボトムシート。発見性と操作効率の両立 |
| iPad TabView Sidebar採用 | iPadOS 18+標準。同一コードベースからの自然な展開。HIG完全準拠 |
| DDP哲学の継承 | 「余白の沈黙」はUI形態に依存しない設計哲学。フィルタUIにもSidebarにも適用可能 |
