## 3. コア機能一覧

### 3.1 機能概要

| # | 機能名 | 概要 | 優先度 |
|---|--------|------|--------|
| 1 | **Share Extension による保存** | ブラウザ・他アプリから1タップでゼロ手間保存 | 必須 |
| 2 | **SM-2 間隔反復** | エビングハウス忘却曲線に基づくスケジューリング。デフォルト「毎朝8時に5件」 | 必須 |
| 3 | **Vision OCR スクショ取込** | スクリーンショット内テキストを自動抽出・保存 | 必須 |
| 4 | **NaturalLanguage 自動タグ** | 日本語テキストを解析してタグを自動生成 | 必須 |
| 5 | **ナレッジマップ** | 保存アイテム間の関連を可視化するグラフUI（標準版: 50件制限） | 推奨 |
| 6 | **QuizMode** | 能動的想起を促す穴埋めクイズ形式の復習モード | 推奨 |
| 7 | **学びジャーナル** | 復習時の気づき・コメントを1行記録 | 推奨 |
| 8 | **Weekly Discovery** | 保存済みタグから関連記事を毎週自動レコメンド（IAP機能） | IAP |

### 3.2 機能詳細

#### Share Extension（ゼロ手間保存）
- iOS標準の共有シートから直接保存
- URL・テキスト・画像をサポート
- 保存後のフリクションを完全排除（確認画面なしで即保存）

#### SM-2 間隔反復
- オープンアルゴリズム（実装ライブラリ豊富）
- デフォルト: 毎朝8時に5件の復習通知
- SM-2パラメータはデフォルト最適化（手動調整UIなし）
- 科学的根拠（エビングハウス忘却曲線）をオンボーディングで可視化説明

#### Vision OCR
- iOS標準 Vision Framework を使用
- スクリーンショット内テキストを自動抽出
- 抽出後のテキストに対して自動タグ生成を適用

#### NaturalLanguage 自動タグ
- iOS標準 NaturalLanguage Framework を使用
- 日本語タグ精度をネイティブAPIで向上
- タグを保存時に自動付与（手動修正可）

#### ナレッジマップ
- Core Data + SwiftUI グラフで実装
- 保存アイテム間の関連を可視化
- 標準版: 50件制限 → ナレッジマップProパック（¥490 IAP）で無制限解放

#### QuizMode
- 保存テキストから穴埋めクイズを自動生成
- 能動的想起（Active Recall）による記憶定着
- 「科学的根拠がある」という訴求で高価格帯を正当化

## 5. 差別化ポイント（ERRCグリッド）

| アクション | 業界標準 | ReCallKitの施策 |
|-----------|---------|---------------|
| **Eliminate（排除）** | Kindle/電子書籍連携、ソーシャル共有、複雑なフォルダ管理、カード手作りフロー | 「保存→自動カード化」のフリクションゼロを徹底。Readwiseの強みエリアに踏み込まない |
| **Reduce（削減）** | 設定の複雑さ（SM-2パラメータ手動調整）、復習頻度カスタマイズ | デフォルト最適化。初期設定は「毎朝8時に5件」の1択プリセット |
| **Raise（引き上げ）** | 日本語対応精度（多くが英語中心）、スクショOCR精度、Share Extension使いやすさ | NaturalLanguage Frameworkで日本語タグ精度向上。Vision OCRでスクショ内テキスト抽出 |
| **Create（創造）** | なし | ナレッジマップ（保存アイテム間の関連可視化）、QuizMode（能動的想起）、学びジャーナル、Weekly Discovery（関連記事推薦） |

## 8. 技術要件

### 使用フレームワーク・API

| カテゴリ | フレームワーク/API | 用途 |
|---------|-----------------|------|
| **保存** | Share Extension（iOS標準） | 他アプリからのゼロ手間保存 |
| **OCR** | Vision Framework（iOS標準） | スクショ内テキスト抽出 |
| **NLP** | NaturalLanguage Framework（iOS標準） | 日本語タグ自動生成 |
| **アルゴリズム** | SM-2（オープンアルゴリズム） | 間隔反復スケジューリング |
| **データ永続化** | Core Data | 保存アイテム・復習履歴管理 |
| **UI** | SwiftUI | グラフUI・ナレッジマップ |
| **通知** | UNUserNotificationCenter | 毎朝8時の復習リマインダー |

### 設計方針

- **サーバーレス設計**: 外部APIゼロ・インフラコストゼロ・保守費ゼロ
- **ローカル完結**: iCloudバックアップはオプション（必須ではない）
- **AI不使用**: オンデバイスのApple標準フレームワークのみで完結

## 9. Expo / React Native 実装時の考慮事項

ReCallKitをExpo Go / React Native で実装する場合の対応方針。

### ネイティブ機能の対応状況

| 機能 | iOS標準 | Expo Go対応 | 代替案・注記 |
|------|--------|:----------:|------------|
| **Share Extension** | ✅ iOS標準 | ⚠️ 要カスタム | `expo-share-extension`（サードパーティ）または EAS Build + ネイティブモジュール。**Expo Go 単体では動作不可**（デベロッパービルド必須） |
| **Vision OCR** | ✅ iOS標準 | ⚠️ 要カスタム | `@ismaelmoreiraa/react-native-vision-camera` + MLKit、または `expo-camera` + `react-native-text-recognition`。完全なVision Framework直接呼び出しはネイティブモジュール要 |
| **NaturalLanguage** | ✅ iOS標準 | ❌ 未対応 | JavaScript側で代替：`natural`（npm）・`kuromoji`（日本語形態素解析）・`compromise`で擬似タグ生成。精度はネイティブより低下 |
| **Core Data** | ✅ iOS標準 | ❌ 未対応 | `expo-sqlite` または `@op-engineering/op-sqlite` で代替。SM-2スケジューリングはJSで実装 |
| **UNUserNotification** | ✅ iOS標準 | ✅ 対応 | `expo-notifications` で対応可能 |
| **SwiftUI グラフ** | ✅ iOS標準 | ❌ 未対応 | `react-native-svg` + D3.js風実装、または `react-native-reanimated` でナレッジマップを再実装 |

### Expo対応の推奨構成

```
推奨ビルド方式: EAS Build（Development Build）
  → Share Extension・Vision OCRなどのネイティブモジュール使用に必須
  → Expo Go単体では Share Extension が動作しないため本番品質には不十分

データ層: expo-sqlite（Core Data代替）
  → SM-2スケジューリングをTypeScript/JSで実装
  → サーバーレス設計を維持できる

OCR: react-native-text-recognition または MLKit
  → Vision Frameworkの完全代替ではないが実用レベル

日本語NLP: kuromoji（npm）
  → ブラウザ/RN両方で動作するJavaScript形態素解析ライブラリ
  → NaturalLanguage Frameworkより精度は劣るが導入コスト低

ナレッジマップ: react-native-svg + カスタムグラフ実装
  → D3.js的な力学グラフ（force-directed graph）をSVGで実現
```

### サーバーレス設計の維持

- 全処理をオンデバイスで完結させる方針は React Native でも維持可能
- 外部APIなし・バックエンドなし → 保守費ゼロを実現
- iCloud同期が必要な場合は `expo-file-system` + CloudKit Bridging を検討

## 10. 改善ポイント TOP3（市場分析より）

1. **初回オンボーディングに「既存保存データのインポート」機能**
   Pocketのエクスポート・URLリストのCSVインポートで初日から大量データを投入。継続率を高める。

2. **Readwise比較ランディング戦略**
   App Store説明文の冒頭に「Readwiseを月$8.99払っていますか？」を直接記載。検索キーワードにReadwiseを想定した対策。

3. **QuizMode（穴埋めクイズ）のデモ動画**
   「読んだ内容が本当に記憶に残る」という科学的根拠（エビングハウス忘却曲線）をビジュアルで説明。高額単価の正当化。
